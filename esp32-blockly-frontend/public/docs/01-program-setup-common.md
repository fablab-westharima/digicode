# プログラム書き込み_共通手順

**最終更新:** 2025-12-06

---

## はじめに

DigiCode（esp32-blockly）では、Blocklyで作成したプログラムを複数種類のマイコンボードに書き込むことができます。本ドキュメントでは、全マイコン共通の準備と基本的な流れを説明します。

---

## 用語の定義

このドキュメントで使用する用語を明確に定義します。

### ファームウェア初期書き込み
DigiCode のホームページから、Arduino C++ または MicroPython を選択して、マイコンに基盤となるシステムソフトウェアを書き込む作業。この作業は **初回のみ必要** です。書き込み後は、同じファームウェアを使い続ける限り、再度書き込む必要はありません。

**ファームウェアの種類:**
- **Arduino C++**: 高速処理が必要な競技用途向け
- **MicroPython**: 初心者向け、学習用途

**どの書き込み方法を使うか決定される:** この時点で、以後のプログラム書き込み時にどの方法を使用するか（USB 有線、OTA など）が決定されます。

### プログラム書き込み
ユーザーが Blockly で作成したプログラムをマイコンに書き込む作業。ファームウェア初期書き込みが既に完了している前提で、アプリケーション部分のみを更新します。ファームウェアは変わらないため、プログラムは何度でも書き込み・更新可能です。

**本ドキュメント内の「ファームウェア書き込み手順」セクションで詳述される手順がこれに該当します。**

**注:** 厳密にはプログラムもファームウェアの一部ですが、本ドキュメントではユーザーが作成するプログラムと、基盤となるシステムソフトウェアを区別するため、この用語を使用しています。

## 対応マイコン一覧

| マイコン族 | ボード数 | 書き込み方法 |
|---------|--------|-----------|
| **RP2040系** | 3種類以上 | USB（1通り） |
| **Arduino系** | 4種類以上 | USB有線 / FTDI等のシリアル（2通り） |
| **ESP系** | 9種類以上 | USB有線 / OTA (WiFi) / Binary直指定（3通り） |

**RP2040系の対応ボード:**
- Raspberry Pi Pico
- Pimoroni Tiny 2040
- Adafruit FeatherS2/S3

**Arduino系の対応ボード:**
- Arduino Uno
- Arduino Nano
- Arduino Nano (Old Bootloader)
- Arduino Leonardo

**ESP系の対応ボード:**
- ESP32
- ESP32-S2 / S3
- ESP32-C2 / C3 / C5 / C6
- ESP8266

## 準備物

### 必須
- パソコン（Windows / Mac / Linux）
- ブラウザ（Chrome推奨）
- 対応マイコンボード

### オプション
- **OTA書き込み用:** WiFi ルータ
- **シリアル接続用:** FTDIなどのUSB-シリアル変換アダプタ

## ファームウェア書き込みの基本流れ

すべてのマイコンは、最初に **ファームウェア** を書き込む必要があります。ファームウェアは、Blocklyで作成したプログラムを実行するための基盤となるソフトウェアです。

### 手順

1. **パソコンと接続**
   - USBケーブルでマイコンをパソコンに接続
   - ボードが認識されることを確認

2. **DigiCode Webで書き込み用バイナリを生成**
   - ブラウザで [DigiCode](https://digicode-frontend.pages.dev) にアクセス
   - ログイン（初回は新規登録）
   - 対応マイコンボードを選択
   - 自動でファームウェアが生成される

3. **書き込みツールで書き込み**
   - 各マイコン別手順を参照

4. **確認**
   - デバイスが立ち上がる（シリアルモニタに `Initializing...` など表示）

## デバイス検出の確認

ファームウェア書き込み後、マイコンが正常に動作しているかの確認方法：

### ESP32系（OTA利用の場合）

```bash
# mDNSでデバイスを検索
dns-sd -B _digicode._tcp local.
```

期待される出力例:
```
Browsing for _digicode._tcp
DATE TIME  Start
DATE TIME  ..
DATE TIME  Instance "esp32-xxxx" in domain local
```

### 全マイコン共通（HTTPで確認）

```bash
# IPアドレスが既知の場合
ping <IPアドレス>

# HTTPサーバーが動作しているか確認
curl http://<IPアドレス>/
```

期待される出力:
```json
{"status":"ok","device":"esp32-xxxx"}
```

## よくある失敗と対処

| 症状 | 原因 | 対処 |
|------|------|------|
| デバイスが認識されない | USB接続不良 | ケーブルを再接続、別のポートを試す |
| 書き込み中にエラー | ドライバ不足 | デバイスマネージャでドライバ確認（Windows） |
| ファームウェア書き込み後も動作しない | デバイス選択ミス | ボード情報を確認してリセット試す |
| OTA検出されない | WiFi未接続 | ファームウェアのWiFi設定確認 |
