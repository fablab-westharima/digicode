# OTA設定ガイド（オプション）

**最終更新:** 2026-04-21

> **このガイドはオプションです。** DigiCodeは**USBケーブルだけで始められます**。まず [はじめかた](./getting-started.md) のUSBフローで使い方に慣れてから、必要と感じたときにこのガイドを参照してください。

> ⚠️ **Windows ユーザーへ**: WiFi OTA は Windows では **上級者向け** です (Bonjour 追加 install + ネットワーク / Firewall 設定が必要)。Windows 初心者の方には:
> - **USB 書き込み** (DigiCode 標準 GUI で案内、追加 install 不要)
> - **Bluetooth (BLE) OTA** (USB 不要、追加 install 不要、初学者向け)
>
> を推奨します。Mac / Linux ユーザーは比較的スムーズに WiFi OTA を試せます。

---

## WiFi OTAとは

WiFi経由でプログラムを書き込む方式（Over-The-Air）です。一度セットアップすれば、以降はUSBケーブルなしでプログラムを更新できます。

### メリット

- USBケーブル接続なしで書き込み可能
- 最速（約15秒）
- 複数台を同時に一括更新可能
- デバイスがケースに入っていても更新可能

### デメリット・注意点

- 初回セットアップ（ファームウェア書き込み＋WiFi設定）が必要
- WiFi接続環境が必要
- PCとESP32が同じネットワークに接続されている必要がある
- Bonjour/mDNSが必要（Windowsでは追加インストール）
- WiFi障害・ネットワーク変更時は書き込み不可

> **中級者以上推奨:** WiFi OTAのトラブル時はUSB接続でのファームウェア再書き込みが必要になる場合があります。

---

## 前提条件

1. **ESP32ボード**（WiFi OTA対応、supportsOta: true）
2. **WiFiルータ**（ESP32が接続できるネットワーク）
3. **DigiCode Finder**（推奨）
   - https://github.com/fablab-westharima/DigiCode-Finder/releases

---

## ステップ1: ファームウェアを書き込む

WiFi OTA / BLEを使うには、最初に1回だけ**ファームウェア**（OTA用の基盤ソフトウェア）を書き込む必要があります。

1. ESP32をUSBケーブルでPCに接続
2. 左メニューの「**ファームウェア書き込み**」をクリック
3. 「INSTALL」ボタンをクリック
4. シリアルポートを選択
5. 書き込み完了を待つ（約1分）

> ファームウェアは一度書き込めば通常は再書き込み不要です。Flash全体消去後は再書き込みが必要です。

---

## ステップ2: ESP32をWiFiに接続する

1. ESP32をUSBケーブルでPCに接続（ファームウェア書き込みと同じ接続）
2. 左メニューの「**WiFi設定**」をクリック
3. シリアルポートを選択して「接続」
4. WiFi設定ダイアログで以下を入力：
   - **SSID**: 接続先WiFiのネットワーク名
   - **パスワード**: WiFiのパスワード
5. 「接続テスト」をクリック
6. 接続成功後、固定IPアドレスが表示される

> 接続テストに成功しないと設定は保存されません。正しいSSIDとパスワードを確認してください。

---

## ステップ3: DigiCode Finderでデバイスを検出

### DigiCode Finderとは

ネットワーク上のDigiCodeデバイスをmDNSで自動検出するデスクトップアプリです（v1.4.1）。

### インストール

**ダウンロード:** https://github.com/fablab-westharima/DigiCode-Finder/releases

| OS | ファイル |
|----|---------|
| Windows | `.exe` |
| macOS | `.dmg` |
| Linux | `.AppImage` |

### Windowsユーザー向け: Bonjourのインストール

DigiCode Finderを起動後、デバイスが検出されない場合はBonjourが必要です。

1. DigiCode Finder のメニュー「Help」→「Install Bonjour (Windows)」
2. Apple公式サイトからBonjour Print Servicesをダウンロード
3. インストール後、PCを再起動

### デバイス検出

1. DigiCode Finderを起動
2. 同じネットワーク上のDigiCodeデバイスが自動検出される
3. デバイス名・IPアドレス・ファームウェアバージョンが表示される

### IPアドレスをコピー

1. 対象デバイスの「選択」ボタンをクリック
2. IPアドレスがクリップボードにコピーされる

### 複数デバイスの選択

1. 複数のデバイスにチェックを入れる
2. 「選択」をクリック（JSON形式で複数IPがコピーされる）

---

## ステップ4: WiFi OTAでプログラム書き込み

### 単一デバイスへの書き込み

1. ブロックエディタでプログラムを作成
2. 「**書き込み**」→「**WiFi OTA**」を選択
3. デバイス選択ダイアログでIPアドレスをペースト
4. 「書き込み開始」をクリック（約15秒）

### 複数デバイスへの一括書き込み

1. DigiCode Finderで複数デバイスを選択してコピー
2. 「書き込み」→「WiFi OTA」
3. デバイス選択ダイアログで複数デバイスが表示される
4. 更新対象にチェックを入れて「一括書き込み開始」

---

## BLE書き込み（オプション）

Bluetooth Low Energy経由で書き込む方式です。WiFi環境がない場所や、ケースに入れたデバイスの更新に便利です。

### BLEのメリット・デメリット

**メリット:**
- WiFi環境が不要
- ケーブル不要
- ケース入りデバイスの更新が可能

**デメリット:**
- WiFi OTAより遅い（約40秒）
- Web Bluetooth対応ブラウザが必要
- ファームウェア書き込みが前提（WiFi OTAと同じ）

### 前提条件

- ファームウェア書き込み済み（ステップ1と同様）
- Web Bluetooth対応ブラウザ（Chrome、Edge）

### 対応ボード

BLE対応（supportsBle: true）のESP32系ボード全般。

### 手順

1. プログラムを作成
2. 「**書き込み**」→「**BLE**」を選択
3. 「デバイスを検索」をクリック
4. ブラウザのBluetooth選択ダイアログで「DigiCode-XXXXXX」を選択
5. 「ペアリング」をクリック
6. 書き込み開始（約40秒）

### BLEトラブルシューティング

| 症状 | 対処 |
|------|------|
| デバイスが見つからない | ESP32を再起動、ブラウザを再起動 |
| ペアリング失敗 | 距離を近づける、他のBLE接続を切断 |
| 書き込みが途中で止まる | ESP32を再起動して再試行 |

---

## デバイス名の設定

複数デバイスを管理する際に役立ちます。

1. USB接続で「WiFi設定」を開く
2. 「デバイス名」欄に新しい名前を入力
3. 「保存」をクリック

**推奨命名規則:**
```
DigiCode-[用途]-[番号]
例: DigiCode-robot-001, DigiCode-class-02
```

---

## トラブルシューティング

### デバイスが検出されない

| 原因 | 対処 |
|------|------|
| WiFi未接続 | USB接続でWiFi設定を確認 |
| 異なるネットワーク | PCとESP32が同じWiFiに接続されているか確認 |
| Bonjour未インストール（Windows） | Bonjour Print Servicesをインストール |
| ファイアウォール | mDNS（ポート5353）を許可 |

### 書き込みが途中で止まる

| 原因 | 対処 |
|------|------|
| WiFi信号が弱い | ESP32をルーターに近づける |
| ネットワーク混雑 | 他の大容量通信を一時停止 |
| タイムアウト | ESP32を再起動して再試行 |

### 書き込み後にデバイスが応答しない

| 原因 | 対処 |
|------|------|
| プログラムのエラー | USB接続でファームウェアを再書き込み |
| WiFi設定の消失 | USB接続でWiFi設定をやり直す |

---

## WiFi設定のリセット

WiFi設定を初期化したい場合：

1. 「ファームウェア書き込み」を開く
2. 「Flash全体を消去（デバッグ用）」を実行
3. ファームウェアを再書き込み
4. WiFi設定をやり直す

---

## セキュリティに関する注意

- 信頼できるネットワーク環境でのみ使用してください
- 公共WiFiでの使用は推奨しません
- 本番環境ではVPNや分離されたネットワークの使用を検討してください
