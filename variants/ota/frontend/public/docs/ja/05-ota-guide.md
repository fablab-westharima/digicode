# OTA 設定ガイド (オプション)

**最終更新:** 2026-05-02

> **このガイドはオプションです。** DigiCode は **USB ケーブルだけで始められます**。まず [はじめかた](./getting-started.md) の USB フローで使い方に慣れてから、必要と感じたときにこのガイドを参照してください。

---

## 🚀 4 ステップで OTA 書き込み完成

WiFi OTA を有効化する全体像。所要時間 ~10 分 (初回のみ)。

1. **ファームウェア書き込み** (USB 経由、~1 分) → [詳細](#ステップ-1-ファームウェアを書き込む)
2. **WiFi 設定** (USB 経由、SSID + パスワード入力、~1 分) → [詳細](#ステップ-2-esp32-を-wifi-に接続する)
3. **デバイス検出** (Mac / Linux は標準 mDNS 自動 / Windows は Bonjour or DigiCode Finder) → [詳細](#ステップ-3-デバイスを検出する)
4. **WiFi OTA で書き込み** (~15 秒、以降ケーブル不要) → [詳細](#ステップ-4-wifi-ota-でプログラム書き込み)

> 💡 **WiFi 環境がない / WiFi 設定が面倒** な場合は **[BLE 書き込み](#ble-書き込みオプション)** がおすすめ (FW 書き込み 1 回だけで使える、WiFi 設定不要)。

---

## ⚠️ Windows ユーザーへの注意

WiFi OTA は Windows では **上級者向け** です (Bonjour 追加 install + ネットワーク / Firewall 設定が必要)。Windows 初心者の方には:

- **USB 書き込み** (DigiCode 標準 GUI で案内、追加 install 不要)
- **Bluetooth (BLE) OTA** (USB 不要、追加 install 不要、初学者向け)

を推奨します。Mac / Linux ユーザーは比較的スムーズに WiFi OTA を試せます。

---

## WiFi OTA とは — メリット・デメリット

WiFi 経由でプログラムを書き込む方式 (Over-The-Air)。一度セットアップすれば、以降は USB ケーブルなしでプログラムを更新できます。

| 観点 | 評価 |
|---|---|
| ✅ ケーブル不要 | USB 接続のたびにケーブルを抜き差しする必要なし |
| ✅ 最速 (~15 秒) | USB (~30 秒) より速い |
| ✅ 複数台同時更新 | 教室での一括更新に有効 |
| ✅ ケース入りデバイス OK | 完成品の更新が可能 |
| ❌ 初回セットアップ必要 | FW 書き込み + WiFi 設定 (~10 分) |
| ❌ WiFi 環境必須 | PC と ESP32 が同 LAN |
| ❌ Bonjour / mDNS 必要 | Windows は追加 install |
| ❌ 障害復旧に USB 必要 | WiFi 設定消失時は USB 経由で再設定 |

> **中級者以上推奨**: WiFi OTA のトラブル時は USB 接続でのファームウェア再書き込みが必要になる場合があります。

---

## 前提条件

1. **ESP32 ボード** (WiFi OTA 対応、`supportsOta: true`)
2. **WiFi ルーター** (ESP32 が接続できるネットワーク、PC と同 LAN)
3. **デバイス検出ツール** (環境別、後述)

---

## ステップ 1: ファームウェアを書き込む

WiFi OTA / BLE を使うには、最初に 1 回だけ **ファームウェア** (OTA receiver) を USB 経由で書き込む必要があります。

1. ESP32 を USB ケーブルで PC に接続
2. 左メニューの **「ファームウェア書き込み」** をクリック
3. **「INSTALL」** ボタンをクリック
4. シリアルポートを選択
5. 書き込み完了を待つ (~1 分)

> 💡 ファームウェアは一度書き込めば通常は再書き込み不要。Flash 全体消去後は再書き込みが必要。

---

## ステップ 2: ESP32 を WiFi に接続する

1. ESP32 を USB ケーブルで PC に接続 (FW 書き込みと同じ接続)
2. 左メニューの **「WiFi 設定」** をクリック
3. シリアルポートを選択して **「接続」**
4. WiFi 設定ダイアログで以下を入力:
   - **SSID**: 接続先 WiFi のネットワーク名
   - **パスワード**: WiFi のパスワード
5. **「接続テスト」** をクリック
6. 接続成功後、固定 IP アドレスが表示される

> ⚠️ 接続テストに成功しないと設定は保存されません。正しい SSID とパスワードを確認してください。

---

## ステップ 3: デバイスを検出する

PC が ESP32 の IP アドレスを取得する手段。OS で必要な手段が異なります。

### Mac / Linux

標準で mDNS 対応のため **追加 install 不要**。DigiCode の WiFi OTA ダイアログでデバイス自動検出が動作します。

### Windows (2 通り)

#### A. Bonjour install (推奨)

Windows 標準では mDNS 非対応のため、Apple の Bonjour Print Services を install:

1. https://support.apple.com/en-us/106380 から `BonjourPSSetup.exe` をダウンロード
2. install → PC 再起動
3. 以降 DigiCode 標準 mDNS 検出が動作

#### B. DigiCode Finder (Bonjour install できない / 制限環境)

[DigiCode Finder](https://github.com/fablab-westharima/DigiCode-Finder/releases) は Windows + Bonjour 未 install 環境向けの mDNS 代替手段デスクトップアプリ。

| OS | DL ファイル |
|---|---|
| Windows | `.exe` |
| (Mac / Linux 版もあるが標準 mDNS で代替可能) | `.dmg` / `.AppImage` |

**使い方**:
1. DigiCode Finder を起動
2. 同じネットワーク上の DigiCode デバイスが自動検出される
3. デバイス名・IP アドレス・ファームウェアバージョンが表示される
4. 対象デバイスの **「選択」** ボタンをクリック → IP アドレスがクリップボードにコピーされる
5. 複数デバイスはチェックを入れて **「選択」** → JSON 形式で複数 IP がコピー

---

## ステップ 4: WiFi OTA でプログラム書き込み

### 単一デバイスへの書き込み

1. ブロックエディタでプログラムを作成
2. **「書き込み」** → **「WiFi OTA」** を選択
3. デバイス選択ダイアログで:
   - mDNS 自動検出が動作する環境 → 一覧から選択
   - それ以外 → IP アドレスを入力 (DigiCode Finder からコピーした値をペースト)
4. **「書き込み開始」** をクリック (~15 秒)

### 複数デバイスへの一括書き込み

1. DigiCode Finder で複数デバイスを選択して JSON コピー
2. **「書き込み」** → **「WiFi OTA」**
3. デバイス選択ダイアログで複数デバイスが表示される
4. 更新対象にチェックを入れて **「一括書き込み開始」**

教室での全台一斉更新に便利。

## BLE 書き込み (オプション)

Bluetooth Low Energy 経由で書き込む方式。WiFi 環境がない場所、ケース入りデバイスの更新、Windows 初学者向け代替手段として有効。

### BLE のメリット・デメリット

| 観点 | 評価 |
|---|---|
| ✅ WiFi 環境不要 | WiFi ルーター / 設定が不要 |
| ✅ ケーブル不要 | USB 接続のたびにケーブル不要 |
| ✅ Windows でも追加 install 不要 | Web Bluetooth は Chrome / Edge 標準 |
| ✅ ケース入りデバイス OK | 完成品の更新が可能 |
| ❌ WiFi OTA より遅い (~40 秒) | 書き込み速度は USB / WiFi OTA より低速 |
| ❌ FW 書き込み必須 | WiFi OTA と同じく初回 USB 必要 |
| ❌ ペアリング距離制限 | 数 m 以内に近づける必要あり |

### 前提条件

- ファームウェア書き込み済み ([ステップ 1](#ステップ-1-ファームウェアを書き込む) と同じ)
- Web Bluetooth 対応ブラウザ (Chrome / Edge)
- 対応ボード: BLE 対応 ESP32 (`supportsBle: true`)

### 手順

1. ブロックエディタでプログラムを作成
2. **「書き込み」** → **「BLE」** を選択
3. **「デバイスを検索」** をクリック
4. ブラウザの Bluetooth 選択ダイアログで **「DigiCode-XXXXXX」** を選択
5. **「ペアリング」** をクリック
6. 書き込み開始 (~40 秒)

### BLE トラブルシューティング

| 症状 | 対処 |
|---|---|
| デバイスが見つからない | ESP32 を再起動、ブラウザを再起動 |
| ペアリング失敗 | 距離を近づける、他の BLE 接続を切断 |
| 書き込みが途中で止まる | ESP32 を再起動して再試行 |

## 詳細・運用

### デバイス名の設定

複数デバイスを管理する際に役立ちます。

1. USB 接続で **「WiFi 設定」** を開く
2. **「デバイス名」** 欄に新しい名前を入力
3. **「保存」** をクリック

**推奨命名規則:**
```
DigiCode-[用途]-[番号]
例: DigiCode-robot-001, DigiCode-class-02
```

### WiFi 設定のリセット

WiFi 設定を初期化したい場合:

1. **「ファームウェア書き込み」** を開く
2. **「Flash 全体を消去 (デバッグ用)」** を実行
3. ファームウェアを再書き込み
4. WiFi 設定をやり直す

### セキュリティに関する注意

- 信頼できるネットワーク環境でのみ使用してください
- 公共 WiFi での使用は推奨しません
- 本番環境では VPN や分離されたネットワークの使用を検討してください

## トラブルシューティング

### デバイスが検出されない

| 原因 | 対処 |
|---|---|
| WiFi 未接続 | USB 接続で WiFi 設定を確認 |
| 異なるネットワーク | PC と ESP32 が同じ WiFi に接続されているか確認 |
| Bonjour 未 install (Windows) | Bonjour Print Services を install または DigiCode Finder を使用 |
| ファイアウォール | mDNS (ポート 5353) を許可 |

### 書き込みが途中で止まる

| 原因 | 対処 |
|---|---|
| WiFi 信号が弱い | ESP32 をルーターに近づける |
| ネットワーク混雑 | 他の大容量通信を一時停止 |
| タイムアウト | ESP32 を再起動して再試行 |

### 書き込み後にデバイスが応答しない

| 原因 | 対処 |
|---|---|
| プログラムのエラー | USB 接続でファームウェアを再書き込み |
| WiFi 設定の消失 | USB 接続で WiFi 設定をやり直す |

詳細は [トラブルシューティング](./troubleshooting.md) を参照。
