# OTA設定ガイド

**最終更新:** 2025-12-28

---

## はじめに

OTA（Over-The-Air）書き込みは、WiFi経由でESP32にプログラムを書き込む方法です。USBケーブルなしでワイヤレスにプログラムを更新できます。

---

## 用語の定義

| 用語 | 説明 |
|------|------|
| **ファームウェア** | DigiCodeの基盤プログラム。初回のみUSBで書き込み |
| **プログラム** | ブロックエディタで作成したユーザープログラム。WiFi OTAで何度でも更新可能 |

詳しくは [プログラム書き込み_共通手順](./01-program-setup-common.md) を参照してください。

---

## OTAの利点と制限

### 利点
- USBケーブルが不要（WiFiだけで更新）
- 複数台を同時に更新可能
- デバイスがケースに入っていても更新可能

### 制限
- 初回はファームウェアをUSBで書き込む必要がある
- WiFi接続環境が必須
- ESP32とPCが同じネットワークに接続されている必要がある

---

## 前提条件

1. **ファームウェアが書き込まれている**
   - 左メニュー「ファームウェア書き込み」からUSBで初期書き込み済み

2. **WiFiルータが利用可能**
   - ESP32が接続できるWiFiネットワーク

3. **DigiCode Finder（推奨）**
   - デバイス検出用デスクトップアプリ
   - https://github.com/fablab-westharima/DigiCode-Finder/releases

---

## ステップ1: ESP32をWiFiに接続する

### USB接続でWiFi設定

1. ESP32をUSBケーブルでPCに接続
2. 左メニューの「**WiFi設定**」（または「ファームウェア書き込み」→「WiFi設定」）をクリック
3. シリアルポートを選択して「接続」
4. WiFi設定ダイアログで以下を入力：
   - **SSID**: 接続先WiFiのネットワーク名
   - **パスワード**: WiFiのパスワード
5. 「接続テスト」をクリック
6. 接続成功後、固定IPアドレスが表示される

> **注意:** WiFi接続テストに成功しないと設定は保存されません。正しいSSIDとパスワードを入力してください。

### 設定内容の確認

接続成功後、以下の情報が表示されます：
- デバイス名
- 割り当てられたIPアドレス
- 接続先SSID

---

## ステップ2: DigiCode Finderでデバイスを検出

### DigiCode Finderとは

ネットワーク上のDigiCodeデバイスをmDNSで自動検出するデスクトップアプリです。

### インストール

1. https://github.com/fablab-westharima/DigiCode-Finder/releases にアクセス
2. お使いのOSに合ったファイルをダウンロード：
   - **Windows**: `.exe` ファイル
   - **macOS**: `.dmg` ファイル
   - **Linux**: `.AppImage` ファイル
3. インストールして起動

### Windowsユーザー向け注意

**Bonjourのインストールが必要です。**

DigiCode Finderを起動後、デバイスが検出されない場合：
1. メニュー「Help」→「Install Bonjour (Windows)」をクリック
2. Apple公式サイトからBonjour Print Servicesをダウンロード
3. インストール後、PCを再起動

### デバイス検出

1. DigiCode Finderを起動
2. 同じネットワーク上のDigiCodeデバイスが自動的に一覧表示される
3. 検出されたデバイスには以下の情報が表示される：
   - デバイス名
   - IPアドレス
   - ファームウェアバージョン

### IPアドレスをコピー

1. 対象デバイスの「選択」ボタンをクリック
2. IPアドレスがクリップボードにコピーされる
3. DigiCode（ブラウザ）で「WiFi OTA」書き込み時にペースト

### 複数デバイスの選択

複数のデバイスを一括で選択することも可能：
1. 複数のデバイスにチェックを入れる
2. 「選択」ボタンをクリック
3. JSON形式で複数のIPアドレスがコピーされる

---

## ステップ3: WiFi OTAでプログラム書き込み

### 単一デバイスへの書き込み

1. DigiCode（ブラウザ）でプログラムを作成
2. 「**書き込み**」ボタンをクリック
3. 「**WiFi OTA**」を選択
4. デバイス選択ダイアログが表示される
5. DigiCode Finderでコピーしたデバイス情報がある場合、自動的に表示される
6. デバイスを選択して「書き込み開始」
7. 進捗バーが表示され、完了を待つ

### 複数デバイスへの一括書き込み

教室など複数のデバイスを同時に更新する場合：

1. DigiCode Finderで複数デバイスを選択してコピー
2. DigiCode（ブラウザ）で「書き込み」→「WiFi OTA」
3. デバイス選択ダイアログで複数デバイスが表示される
4. 更新対象にチェックを入れる（全選択も可能）
5. 「一括書き込み開始」をクリック
6. 各デバイスの進捗が表示される

---

## ステップ4: 動作確認

書き込み完了後、ESP32は自動的に再起動してプログラムが実行されます。

### 確認方法

1. **動作確認**: 実際にプログラムが動作しているか確認
2. **シリアルモニタ**: USB接続時はシリアル出力で確認可能
3. **DigiCode Finder**: デバイスが再検出されれば正常

---

## デバイス名の設定

デバイス名は、複数のデバイスを管理する際に重要です。

### デバイス名の変更方法

1. USB接続で「WiFi設定」を開く
2. 「デバイス名」欄に新しい名前を入力
3. 「保存」をクリック

### 推奨の命名規則

```
DigiCode-[用途]-[番号]

例:
- DigiCode-robot-001
- DigiCode-sensor-lab-a
- DigiCode-class-02
```

## トラブルシューティング

### デバイスが検出されない

| 原因 | 対処 |
|------|------|
| WiFi未接続 | USB接続でWiFi設定を確認 |
| 異なるネットワーク | PCとESP32が同じWiFiに接続されているか確認 |
| Bonjour未インストール（Windows） | Bonjour Print Servicesをインストール |
| ファイアウォール | mDNS（ポート5353）を許可 |

### 書き込みが途中で止まる

| 原因 | 対処 |
|------|------|
| WiFi信号が弱い | ESP32をルーターに近づける |
| ネットワーク混雑 | 他の大容量通信を一時停止 |
| タイムアウト | ESP32を再起動して再試行 |

### 書き込み後にデバイスが応答しない

| 原因 | 対処 |
|------|------|
| プログラムのエラー | USB接続でファームウェアを再書き込み |
| WiFi設定の消失 | USB接続でWiFi設定をやり直す |

---

## WiFi設定のリセット

WiFi設定を初期化したい場合：

1. 「ファームウェア書き込み」を開く
2. 「Flash全体を消去（デバッグ用）」を実行
3. ファームウェアを再書き込み
4. WiFi設定をやり直す

---

## セキュリティに関する注意

- 信頼できるネットワーク環境でのみ使用してください
- 公共WiFiでの使用は推奨しません
- 本番環境ではVPNや分離されたネットワークの使用を検討してください
