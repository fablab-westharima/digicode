# トラブルシューティング

**最終更新:** 2026-04-21

DigiCode でよくある問題と解決方法をまとめました。

---

## プログラム書き込み関連

### ESP32 が認識されない

**症状:** シリアルポート選択ダイアログに ESP32 が表示されない

**解決策:**

1. USB ケーブルが**データ通信対応**であることを確認（充電専用ケーブルは不可）
2. ESP32 の USB ドライバーをインストール
   - **CP2102**: https://www.silabs.com/developers/usb-to-uart-bridge-vcp-drivers
   - **CH340**: http://www.wch.cn/downloads/CH341SER_ZIP.html
3. 別の USB ポートを試す
4. ESP32 の電源を入れ直す（USB ケーブルを抜き差し）
5. ブラウザを Chrome / Edge に切り替え（Web Serial API 対応ブラウザ必須）

### 書き込みエラーが発生する

**症状:** 「Failed to connect」などのエラーが表示される

**解決策:**

1. ESP32 の **BOOT ボタン**を押しながら書き込みを開始
2. 書き込み中は BOOT ボタンを押し続ける
3. シリアルモニタなど、他のアプリがポートを使用していないか確認
4. ブラウザを再起動して再試行

### コンパイル枠を使い切った

**症状:** 「本月のコンパイル上限に到達しました」と表示される

**解決策:**

1. 翌月まで待つ
2. 上位プラン（Lite / Pro / Enterprise）にアップグレード
3. [ローカルコンパイルサーバー](./local-compile-server.md) を使用すればクラウド枠を消費しません

## WiFi OTA 関連

### WiFi OTA でデバイスが見つからない

**症状:** DigiCode Finder でデバイスが検出されない

**解決策:**

1. ESP32 と PC が**同じ WiFi ネットワーク**に接続されているか確認
2. **Bonjour Print Services**（Windows のみ）がインストールされているか確認
3. ファイアウォールが mDNS（ポート 5353）をブロックしていないか確認
4. ESP32 のシリアルモニタで、WiFi 接続成功と IP アドレスを確認
5. ESP32 を再起動（RESET ボタン）

### OTA 更新が途中で止まる

**症状:** 更新が 20% 程度で停止する

**解決策:**

1. WiFi 信号が弱い可能性 → ESP32 をルーターに近づける
2. ESP32 を再起動してから再試行
3. USB 経由でファームウェアを再書き込みしてから、WiFi OTA を再試行

### WiFi OTA 書き込み後にデバイスが応答しない

**症状:** 書き込みは成功したが、プログラムが動作しない

**解決策:**

1. USB 経由でファームウェアを再書き込み
2. 「Flash全体を消去（デバッグ用）」→ ファームウェア書き込み → WiFi 設定

## BLE 関連

### BLE でデバイスが見つからない

**症状:** 「デバイスを検索」ボタンを押してもデバイスが表示されない

**解決策:**

1. **ブラウザ対応** - Chrome / Edge を使用しているか確認（Web Bluetooth 必須）
2. **ボード対応** - ESP32-S2 は BLE 非対応、RP2040 Pico W も BLE 非対応
3. ESP32 を再起動
4. ブラウザを再起動

### BLE ペアリングが失敗する

**症状:** デバイスは見つかるがペアリングに失敗

**解決策:**

1. ESP32 をパソコンの近くに配置
2. 他の BLE 接続（マウス・キーボード等）を一時切断
3. ESP32 を再起動して再試行

### macOS で古いデバイス名が表示される

**症状:** デバイス名を変更したのに macOS のブラウザで古い名前が表示される

**解決策:** macOS の BLE キャッシュが原因の OS レベル問題で、DigiCode 側では修正できません。Windows では正常動作します。Mac では影響を受ける場合があります。

## プログラム動作関連

### プログラムが動かない

**症状:** 書き込みは成功するが、プログラムが動作しない

**解決策:**

1. ESP32 の RESET ボタンを押して再起動
2. シリアルモニタでエラーメッセージを確認
3. ピン番号が正しいか確認（ボードによって使用可能なピンが異なります）
4. センサー・モーターの配線を再確認

### LED が点滅しない

**症状:** Lチカプログラムを書き込んだが、LED が点滅しない

**解決策:**

1. **GPIO2（内蔵 LED）の場合**:
   - 一部のボードには内蔵 LED がありません
   - 外部 LED を接続して試す
2. **外部 LED の場合**:
   - 極性（＋−）を確認
   - 抵抗（330Ω〜1kΩ）を直列に接続
   - ピン番号を確認

## クラス機能関連

### 招待 URL からクラスに参加できない

**症状:** 招待 URL を開いてもクラスに参加できない

**解決策:**

1. DigiCode にログインしているか確認（未ログインの場合はログイン後に URL を再度開く）
2. 招待 URL が期限切れでないか（教員に確認）
3. 同じ招待 URL で既に別アカウントで参加していないか

### 課題が表示されない

**症状:** クラスに参加しているが課題が表示されない

**解決策:**

1. ページを再読み込み
2. クラスサーバー（`class.digital-fab.jp`）の稼働状況を確認
3. 教員に課題が配布済みか確認

### 答案が提出できない

**症状:** 答案送信ボタンを押してもエラー

**解決策:**

1. ネットワーク接続を確認
2. クラスサーバーが一時的にダウンしている可能性 → 時間を置いて再試行
3. Blockly XML が大きすぎないか（通常は問題ない）

## プラン・決済関連

### プラン変更できない

**症状:** Stripe カスタマーポータルが開かない、またはエラー

**解決策:**

1. ポップアップブロッカーが有効になっていないか確認
2. ブラウザの Cookie が有効になっているか確認
3. Stripe のサーバー稼働状況を確認

### 招待アカウントで自分でプラン契約したら？

**症状:** ファブラボ西播磨から Enterprise 相当の権限を付与されているが、自分で Pro を契約した場合

**解決策:** 自分で契約した場合、招待による権限付与は**解除**されます。以降は自分の契約プランが適用されます。確認ダイアログで内容を確認してから契約してください。

## ブラウザ関連

### Blockly エディタが表示されない

**症状:** エディタページが真っ白、またはブロックが表示されない

**解決策:**

1. ブラウザのキャッシュをクリア
2. ブラウザを最新版にアップデート
3. 別のブラウザ（Chrome、Edge）を試す
4. JavaScript が有効になっているか確認

### コンパイルエラーが表示される

**症状:** 「コンパイルエラー」ダイアログが表示される

**解決策:**

1. エラーメッセージを確認
2. ブロックの接続が正しいか確認
3. 必要なパラメータがすべて入力されているか確認
4. サンプルプロジェクトから試して、問題を切り分ける

---

## その他

### プロジェクトが保存できない

**症状:** 保存ボタンを押してもエラーが発生する

**解決策:**

1. ログインしているか確認（ゲストはローカル保存のみ）
2. インターネット接続を確認
3. ブラウザのローカルストレージが有効か確認
4. ブラウザを再起動

### サンプルプロジェクトが読み込めない

**症状:** サンプルを選択してもエディタに反映されない

**解決策:**

1. ページをリロード
2. 一度ログアウトしてから再ログイン
3. 別のサンプルを試す

## サポート

上記で解決しない場合は、以下の情報を添えて [GitHub Issues](https://github.com/fablab-westharima/DigiCode/issues) で報告してください：

- 使用しているブラウザとバージョン
- OS
- 使用しているプラン
- ESP32 のボード名
- 書き込み方式（USB / WiFi OTA / BLE）
- エラーメッセージのスクリーンショット
- 再現手順
