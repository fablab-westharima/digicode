# トラブルシューティング

DigiCodeでよくある問題と解決方法をまとめました。

## ファームウェア書き込み関連

### ESP32が認識されない

**症状:** シリアルポートに何も表示されない

**解決策:**
1. USBケーブルが**データ転送対応**であることを確認（充電専用ケーブルは不可）
2. ESP32のUSBドライバーをインストール
   - CP2102ドライバー: https://www.silabs.com/developers/usb-to-uart-bridge-vcp-drivers
   - CH340ドライバー: http://www.wch.cn/downloads/CH341SER_ZIP.html
3. 別のUSBポートを試す
4. ESP32の電源を入れ直す（USBケーブルを抜き差し）

### 書き込みエラーが発生する

**症状:** 「Failed to connect」などのエラーが表示される

**解決策:**
1. ESP32の**BOOTボタン**を押しながら書き込みを開始
2. 書き込み中はBOOTボタンを押し続ける
3. シリアルモニターなど、他のアプリがポートを使用していないか確認
4. ブラウザを再起動して再試行

## プログラム動作関連

### プログラムが動かない

**症状:** 書き込みは成功するが、プログラムが動作しない

**解決策:**
1. ESP32のRESETボタンを押して再起動
2. シリアルモニターでエラーメッセージを確認
3. ピン番号が正しいか確認（ボードによって使用可能なピンが異なります）
4. センサー・モーターの配線を再確認

### LEDが点滅しない

**症状:** Lチカプログラムを書き込んだが、LEDが点滅しない

**解決策:**
1. GPIO2（内蔵LED）の場合:
   - 一部のボードには内蔵LEDがありません
   - 外部LEDを接続して試す
2. 外部LEDの場合:
   - 極性（＋−）を確認
   - 抵抗（330Ω〜1kΩ）を直列に接続
   - ピン番号を確認

## WiFi OTA関連

### WiFi OTA更新ができない

**症状:** デバイスがスキャンで見つからない、または接続できない

**解決策:**
1. ESP32とPCが**同じWiFiネットワーク**に接続されているか確認
2. ファイアウォールがmDNS（ポート5353）をブロックしていないか確認
3. ESP32のシリアルモニターで、WiFi接続成功を確認
4. ESP32を再起動（RESETボタン）

### OTA更新が途中で止まる

**症状:** 更新が20%程度で停止する

**解決策:**
1. WiFi信号が弱い可能性 → ESP32をルーターに近づける
2. ESP32を再起動してから再試行
3. USB経由で書き込んでから、WiFi OTAを再度試す

## ブラウザ関連

### Blocklyエディタが表示されない

**症状:** エディタページが真っ白、またはブロックが表示されない

**解決策:**
1. ブラウザのキャッシュをクリア
2. ブラウザを最新版にアップデート
3. 別のブラウザ（Chrome、Edge、Safari）を試す
4. JavaScriptが有効になっているか確認

### コンパイルエラーが表示される

**症状:** 「コンパイルエラー」ダイアログが表示される

**解決策:**
1. エラーメッセージを確認
2. ブロックの接続が正しいか確認
3. 必要なパラメータがすべて入力されているか確認
4. サンプルプロジェクトから試して、問題を切り分ける

## その他

### プロジェクトが保存できない

**症状:** 保存ボタンを押してもエラーが発生する

**解決策:**
1. ログインしているか確認
2. インターネット接続を確認
3. ブラウザのローカルストレージが有効か確認
4. ブラウザを再起動

### サンプルプロジェクトが読み込めない

**症状:** サンプルを選択してもエディタに反映されない

**解決策:**
1. ページをリロード
2. 一度ログアウトしてから再ログイン
3. 別のサンプルを試す

## サポート

上記で解決しない場合は、以下の情報を添えてGitHub Issueで報告してください：

- 使用しているブラウザとバージョン
- ESP32のボード名（例: ESP32-DevKitC）
- エラーメッセージのスクリーンショット
- 再現手順

**GitHub Issues:** https://github.com/fablab-westharima/DigiCode/issues
