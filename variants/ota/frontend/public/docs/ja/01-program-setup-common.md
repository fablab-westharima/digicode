# プログラム書き込み_共通手順

**最終更新:** 2025-12-28

---

## はじめに

DigiCodeでは、Blocklyで作成したプログラムを複数種類のマイコンボードに書き込むことができます。本ドキュメントでは、全マイコン共通の準備と基本的な流れを説明します。

---

## 用語の定義

このドキュメントで使用する用語を明確に定義します。

### ファームウェア書き込み

DigiCodeの基盤となるシステムソフトウェアを書き込む作業。この作業は**初回のみ必要**です。

| 項目 | 内容 |
|------|------|
| **何を書き込むか** | DigiCodeの基盤プログラム（Arduino C++） |
| **書き込み方法** | USB経由のみ |
| **書き込み頻度** | 初回のみ（Flash消去後の再設定時を除く） |
| **UI上の場所** | 左メニュー「ファームウェア書き込み」 |

### プログラム書き込み

ユーザーがBlocklyで作成したプログラムをマイコンに書き込む作業。ファームウェア書き込みが完了している前提で、何度でも書き込み・更新可能です。

| 項目 | 内容 |
|------|------|
| **何を書き込むか** | ブロックエディタで作成したユーザープログラム |
| **書き込み方法** | WiFi OTA / BLE / USB |
| **書き込み頻度** | 何度でも（プログラム変更のたびに） |
| **UI上の場所** | エディタ画面「書き込み」ボタン |

| マイコン族 | ボード数 | 書き込み方法 |
|---------|--------|-----------|
| **ESP32系** | 9種類以上 | WiFi OTA / BLE / USB |
| **RP2040系** | 3種類以上 | USB |
| **Arduino系** | 4種類以上 | USB |

### ESP32系の対応ボード

### RP2040系の対応ボード

### Arduino系の対応ボード
- Arduino Uno
- Arduino Nano

- Arduino Leonardo

## 3つの書き込み方式

### WiFi OTA（Over-The-Air）

WiFi経由でワイヤレスにプログラムを書き込む方式。

| 項目 | 内容 |
|------|------|
| **メリット** | 高速、ケーブル不要 |
| **デメリット** | 初回はファームウェア書き込みが必要 |
| **対応ボード** | ESP32系 |
| **必要なもの** | DigiCode Finder（デバイス検出アプリ） |

### BLE（Bluetooth Low Energy）

Bluetooth経由でプログラムを書き込む方式。

| 項目 | 内容 |
|------|------|
| **メリット** | ケーブル不要、WiFiがなくても使用可能 |
| **デメリット** | WiFi OTAより遅い |
| **対応ボード** | ESP32（S2除く） |
| **必要なもの** | Web Bluetooth対応ブラウザ（Chrome、Edge） |

### USB直接書き込み

USBケーブルで接続して書き込む方式。

| 項目 | 内容 |
|------|------|
| **メリット** | 確実、どの環境でも動作 |
| **デメリット** | ケーブル接続が必要 |
| **対応ボード** | 全ボード |
| **必要なもの** | USBケーブル、ドライバー |

## 準備物

### 必須
- パソコン（Windows / Mac / Linux）
- ブラウザ（Chrome、Edge推奨）
- 対応マイコンボード
- USBケーブル（初回ファームウェア書き込み用）

### オプション
- **WiFi OTA用:** WiFiルータ、DigiCode Finder
- **BLE用:** Web Bluetooth対応ブラウザ

## ファームウェア書き込みの基本流れ

すべてのマイコンは、最初に**ファームウェア**を書き込む必要があります。

### 手順

1. **PCとUSB接続**
   - USBケーブルでマイコンをPCに接続
   - ボードが認識されることを確認

2. **DigiCodeでファームウェア書き込み**
   - 左メニュー「ファームウェア書き込み」をクリック
   - 「INSTALL」ボタンをクリック
   - シリアルポートを選択
   - 書き込み完了を待つ（約1分）

3. **確認**
   - デバイスが再起動して正常に動作

### ESP32でWiFi OTAを使用する場合

ファームウェア書き込み後に追加設定が必要：

1. 「WiFi設定」をクリック
2. シリアルポートを選択して接続
3. SSIDとパスワードを入力
4. 「接続テスト」をクリック
5. 接続成功後、固定IPアドレスが表示される

---

## デバイス検出の確認

### ESP32系（WiFi OTA利用時）

**DigiCode Finderを使用（推奨）:**

1. DigiCode Finderをダウンロード
   - https://github.com/fablab-westharima/DigiCode-Finder/releases
2. アプリを起動
3. 同じネットワーク上のDigiCodeデバイスが自動検出される

### 全マイコン共通（USB接続時）

1. ESP32をUSB接続
2. サイドバーの「シリアルモニタ」をクリック
3. ポートを選択して接続
4. 出力が表示されれば正常

---

## よくある失敗と対処

| 症状 | 原因 | 対処 |
|------|------|------|
| デバイスが認識されない | USB接続不良 | ケーブルを再接続、別のポートを試す |
| 書き込み中にエラー | ドライバ不足 | USBドライバをインストール |
| ファームウェア書き込み後も動作しない | デバイス選択ミス | ボード情報を確認してリセット |
| WiFi OTAで検出されない | WiFi未接続 | USB接続でWiFi設定を確認 |
