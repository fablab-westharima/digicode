# プログラム書き込み — 共通手順

**最終更新:** 2026-05-02

DigiCode は **ESP32 系専用** のブロックプログラミング環境です。本ドキュメントでは「どの書き込み方式を選ぶか」と用語の概要を説明します。

---

## 🚀 どの方式を使う？ — 30 秒で決まる

| あなたの状況 | 推奨方式 |
|---|---|
| **初めて使う** / **トラブルなく確実に書き込みたい** | 🥇 **USB 直接書き込み** ([詳細](#1-usb-直接書き込み-基本-推奨)) |
| ESP32 をケース内に入れる / 毎回 USB 接続が面倒 / 複数台同時に書き込みたい | 🥈 **WiFi OTA** ([詳細](./05-ota-guide.md)) |
| WiFi 環境がない / WiFi 設定が面倒 | 🥉 **BLE 書き込み** ([詳細](./05-ota-guide.md)) |
| クラウドコンパイル枠を超えた / オフラインで使いたい / 高速化したい | ⚡ **ローカルコンパイル + 上記いずれか** ([詳細](./local-compile-server.md)) |

> 💡 **迷ったら USB 直接書き込みから始めてください**。WiFi OTA / BLE は USB で「ファームウェア書き込み」を一度行ってから使えるオプション機能です。

---

## 4 つの書き込み方式 — 比較表

| 方式 | 速度 | ケーブル | 事前準備 | 対応ボード |
|---|---|---|---|---|
| **🥇 USB 直接** | ~30 秒 | 必要 | USB ドライバ | 全ボード |
| **🥈 WiFi OTA** | ~15 秒 (最速) | 不要 | FW 書き込み + WiFi 設定 (初回のみ) | ESP32 系全般 (`supportsOta: true`) |
| **🥉 BLE** | ~40 秒 | 不要 | FW 書き込み (初回のみ) | BLE 対応 ESP32 (`supportsBle: true`) |
| **⚡ ローカルコンパイル** | (組合わせ用、速度は上記書き込み方式に依存、cache HIT で ~1 ms) | — | Docker | 全ボード (組合せ可) |

---

## 対応マイコン

DigiCode は **ESP32 系専用** です。

| カテゴリ | ボード例 | USB | WiFi OTA | BLE |
|---|---|:---:|:---:|:---:|
| **Generic ESP32 系** | ESP32 / S3 / C3 / C6 | ○ | ○ | ○ |
| **M5Stack 系** | M5Stack Basic / M5StickC Plus / ATOM / M5Stamp 等 | ○ | ○ | ○ |
| **XIAO ESP32 系** | XIAO ESP32C3 / S3 / C6 | ○ | ○ | ○ |

> **非対応:** ESP8266、Arduino Uno/Nano、RP2040 系は対応対象外です。

---

## 書き込みの基本フロー (USB)

1. **ESP32 を USB ケーブルで PC に接続**
2. ブロックエディタでプログラムを作成 (またはサンプル読み込み)
3. 右上「**書き込み**」ボタン → 「**USB**」を選択
4. ブラウザのシリアルポート選択ダイアログでポートを選択
5. 書き込み完了後、ESP32 が自動再起動してプログラム実行

📘 **完全な初回手順は** [はじめかた](./getting-started.md) **を参照** (ドライバ install + LED 点滅サンプルまで)。

## 詳細・概念

### 用語: プログラム書き込み vs ファームウェア書き込み

DigiCode では「プログラム書き込み」と「ファームウェア書き込み」は **異なる操作** です。

#### プログラム書き込み (通常の操作)

ブロックエディタで作成したプログラムをマイコンに書き込む操作。

| 項目 | 内容 |
|---|---|
| **何を書き込むか** | ユーザーがブロックで作ったプログラム |
| **書き込み方法** | USB (基本) / WiFi OTA / BLE |
| **書き込み頻度** | 何度でも (プログラムを変えるたびに) |
| **UI 上の場所** | エディタ画面「書き込み」ボタン |

#### ファームウェア書き込み (WiFi OTA / BLE 使用時のみ)

WiFi OTA または BLE を使うために必要な基盤ソフトウェア (OTA receiver) を書き込む操作。**USB だけでプログラムを書き込む場合は不要** です。

| 項目 | 内容 |
|---|---|
| **何を書き込むか** | OTA 用の基盤ソフトウェア (ハードウェア制御の最小限の基盤命令) |
| **書き込み方法** | USB 経由のみ |
| **書き込み頻度** | WiFi OTA / BLE 使用時の初回のみ |
| **UI 上の場所** | 左メニュー「ファームウェア書き込み」 |

### 4 つの書き込み方式 詳細

#### 1. USB 直接書き込み (基本・推奨)

最も確実・どの環境でも動作。USB ドライバ (CP2102 / CH340) が事前に必要。詳細は [はじめかた](./getting-started.md) と [ESP32 書き込みガイド](./04-program-setup-esp32.md)。

#### 2. WiFi OTA (オプション)

ケーブル不要・最速・複数台同時更新可。WiFi ルーター必須。Windows + Bonjour 未 install 環境では [DigiCode Finder](https://github.com/fablab-westharima/DigiCode-Finder) を mDNS 代替として使用可能 (Mac / Linux は標準で mDNS 対応)。詳細は [OTA 設定ガイド](./05-ota-guide.md)。

> ⚠️ **Windows ユーザーの方へ**: WiFi OTA 構築は Windows では上級者向けです。初学者は USB / BLE 書き込みを推奨します。詳細は [ローカルコンパイルサーバー § Windows](./local-compile-server.md#-まず-推奨度と前提知識) を参照。

#### 3. BLE 書き込み (オプション)

ケーブル不要・WiFi 環境不要。Web Bluetooth 対応ブラウザ (Chrome / Edge) 必須。詳細は [OTA 設定ガイド](./05-ota-guide.md)。

#### 4. ローカルコンパイルサーバー (高速化オプション)

クラウドコンパイル枠を使わない・オフライン利用可。Docker (または OrbStack / Rancher Desktop / Podman) 必須。書き込み方式 (USB / WiFi OTA / BLE) と直交、組み合わせ可能。詳細は [ローカルコンパイルサーバー](./local-compile-server.md)。

### 準備物

#### 必須
- パソコン (Windows / Mac / Linux)
- Web ブラウザ (Chrome / Edge 推奨)
- 対応 ESP32 ボード
- USB ケーブル (データ通信対応)
- USB ドライバ (CP2102 または CH340)

#### オプション
- **WiFi OTA 用**: WiFi ルーター (Windows + Bonjour 未 install 環境では DigiCode Finder)
- **BLE 用**: Web Bluetooth 対応ブラウザ (Chrome / Edge)
- **ローカルコンパイル用**: Docker 環境

### WiFi OTA / BLE を使いたい場合

初回のみ以下のセットアップが必要です:

1. USB で ESP32 を接続
2. 左メニュー「**ファームウェア書き込み**」→「**INSTALL**」を実行
3. (WiFi OTA の場合) 「**WiFi 設定**」で SSID とパスワードを設定
4. 以降は USB 接続なしで WiFi OTA / BLE で書き込み可能

→ 詳細: [OTA 設定ガイド](./05-ota-guide.md)

## よくある失敗と対処

| 症状 | 原因 | 対処 |
|---|---|---|
| ポートが表示されない | USB ドライバ未 install | CP2102 または CH340 ドライバを install + PC 再起動 |
| 書き込みエラー | USB ケーブルが充電専用 | データ通信対応ケーブルに交換 |
| 書き込み中にタイムアウト | 接続不安定 | 別の USB ポート / 別のケーブルを試す |
| BOOT モードに入れない | ボード固有の手順 | BOOT / BOOTSTRAP ボタンを押しながら書き込み開始 |
| WiFi OTA で検出されない | WiFi 未接続 / FW 未書き込み | USB 接続でファームウェア書き込みと WiFi 設定を確認 |

詳細は [トラブルシューティング](./troubleshooting.md) を参照。
