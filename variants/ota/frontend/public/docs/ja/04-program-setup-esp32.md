# プログラム書き込み_ESP32系

**最終更新:** 2025-12-28

---

## 対応ボード

- ESP32
- ESP32-S2 / S3
- ESP32-C2 / C3 / C5 / C6
- ESP8266
- その他ESP-IDF対応ボード

---

## ESP32の特徴

ESP32はWiFi/Bluetoothを搭載した高機能マイコンです。

- **言語:** Arduino C++
- **接続方法:** WiFi OTA、BLE、USB
- **ブートローダー:** ROMに固定（通常は置き換え不要）

---

## 用語の定義

| 用語 | 説明 |
|------|------|
| **ファームウェア** | DigiCodeの基盤プログラム。初回のみUSBで書き込み |
| **プログラム** | ブロックエディタで作成したユーザープログラム。何度でも更新可能 |

詳しくは [プログラム書き込み_共通手順](./01-program-setup-common.md) を参照してください。

---

## 3つの書き込み方式

ESP32では3つの書き込み方式が使用可能です。

| 方式 | 用途 | 対応ボード | 特徴 |
|------|------|-----------|------|
| **WiFi OTA** | 通常の更新 | ESP32全般 | 最速、ケーブル不要 |
| **BLE** | AP制限環境・ケース入りデバイス | ESP32（S2除く） | ケーブル不要、WiFiなしでも可能 |
| **USB** | 初回設定・確実な書き込み | 全ボード | どの環境でも確実に動作 |

---

## 方式1: WiFi OTA書き込み（推奨）

ワイヤレスで最も高速な書き込み方式です。

### 前提条件

- ファームウェアがUSB経由で書き込み済み
- ESP32がWiFiに接続済み
- DigiCode Finder（デスクトップアプリ）

### 手順

1. **DigiCode Finderを起動**
   - ネットワーク上のDigiCodeデバイスが自動検出される
   - ダウンロード: https://github.com/fablab-westharima/DigiCode-Finder/releases

2. **デバイスを選択してIPをコピー**
   - 対象デバイスの「選択」ボタンをクリック
   - IPアドレスがクリップボードにコピーされる

3. **DigiCode（ブラウザ）で書き込み**
   - プログラムを作成
   - 「書き込み」ボタンをクリック
   - 「WiFi OTA」を選択
   - デバイス選択ダイアログでIPをペースト
   - 「書き込み開始」

詳細は [OTA設定ガイド](./05-ota-guide.md) を参照してください。

---

## 方式2: BLE書き込み

Bluetooth Low Energyを使用した書き込み方式です。WiFiが使えない環境やケースに入ったデバイスの更新に便利です。

### 前提条件

- ファームウェアがUSB経由で書き込み済み
- Web Bluetooth対応ブラウザ（Chrome、Edge）

### 対応ボード

- ESP32（標準）
- ESP32-C3
- ESP32-S3

> **注意:** ESP32-S2はBLEに対応していません。

### 手順

1. **DigiCode（ブラウザ）でプログラムを作成**

2. **「書き込み」→「BLE」を選択**

3. **Bluetoothスキャン開始**
   - 「デバイスを検索」ボタンをクリック
   - ブラウザのBluetooth選択ダイアログが表示される

4. **デバイスを選択**
   - 「DigiCode-XXXXXX」という名前のデバイスを選択
   - 「ペアリング」をクリック

5. **書き込み開始**
   - 進捗バーが表示される
   - 完了を待つ（WiFi OTAより遅い）

### トラブルシューティング

| 症状 | 対処 |
|------|------|
| デバイスが見つからない | ESP32を再起動、ブラウザを再起動 |
| ペアリング失敗 | 距離を近づける、他のBLE接続を切断 |
| 書き込みが途中で止まる | ESP32を再起動して再試行 |

---

## 方式3: USB直接書き込み

最も確実な書き込み方式です。初回セットアップやトラブル時に使用します。

### 前提条件

- USBケーブル（データ通信対応）
- USBドライバー（CP2102またはCH340）

### 手順

1. **ESP32をUSBケーブルでPCに接続**

2. **DigiCode（ブラウザ）でプログラムを作成**

3. **「書き込み」→「USB」を選択**

4. **シリアルポートを選択**
   - ブラウザのシリアルポート選択ダイアログが表示される
   - ESP32が接続されているポートを選択

5. **書き込み開始**
   - 4つのファイルが順次書き込まれる（fullPackageモード）
   - bootloader.bin → partitions.bin → boot_app0.bin → firmware.bin

### USBドライバーのインストール

ESP32が認識されない場合は、USBドライバーのインストールが必要です。

**CP2102チップ搭載ボード:**
- https://www.silabs.com/developers/usb-to-uart-bridge-vcp-drivers

**CH340チップ搭載ボード:**
- http://www.wch.cn/downloads/CH341SER_ZIP.html

### トラブルシューティング

| 症状 | 対処 |
|------|------|
| ポートが表示されない | USBドライバーをインストール |
| 書き込み失敗 | BOOTボタンを押しながら書き込み開始 |
| タイムアウトエラー | 別のUSBポートを試す |

## ファームウェア初期書き込み

新しいESP32に初めてDigiCodeを使用する場合、ファームウェアの書き込みが必要です。

### 手順

1. 左メニューの「**ファームウェア書き込み**」をクリック
2. ESP32をUSBケーブルでPCに接続
3. 「INSTALL」ボタンをクリック
4. シリアルポートを選択
5. 書き込み完了を待つ（約1分）

### WiFi設定（OTA用）

ファームウェア書き込み後、WiFi OTAを使用するには：

1. 「WiFi設定」をクリック
2. シリアルポートを選択して接続
3. SSIDとパスワードを入力
4. 「接続テスト」をクリック
5. 成功後、固定IPアドレスが表示される

## シリアルモニタで動作確認

プログラムのデバッグには、シリアルモニタが便利です。

### ブラウザでの確認

DigiCodeにはシリアルモニタ機能が内蔵されています：

1. ESP32をUSB接続
2. サイドバーの「シリアルモニタ」をクリック
3. ポートを選択して「接続」
4. ボーレート: 115200

### Arduino IDEでの確認

1. ツール → シリアルモニタを開く
2. ボーレート: 115200
3. ESP32からの出力が表示される

## 書き込み方式の選び方

| 状況 | 推奨方式 |
|------|---------|
| 通常の更新（最速） | WiFi OTA |
| WiFiが使えない環境 | BLE |
| 初回セットアップ | USB |
| トラブル時の復旧 | USB |
| ケースに入ったデバイス | BLE or WiFi OTA |
| 教室・WS（多数のデバイス） | WiFi OTA（一括更新） |
