## 用語

| 用語 | 説明 |
|------|------|
| **プログラム書き込み** | ブロックエディタで作成したプログラムを書き込む操作（毎回） |
| **ファームウェア書き込み** | WiFi OTA / BLE用の基盤ソフトウェアを書き込む操作（**WiFi OTA / BLE使用時の初回のみ**） |

詳しくは [共通手順](./01-program-setup-common.md) を参照してください。

## 方式1: USB直接書き込み（基本・推奨）

最も確実な書き込み方式です。初心者からトラブル復旧まで幅広く使えます。

### 前提条件

- USBケーブル（データ通信対応）
- USBドライバー（CP2102またはCH340）

### USBドライバーのインストール

ESP32が認識されない場合はドライバーのインストールが必要です。

**CP2102搭載ボード（多くのESP32-DevKitC、M5StampS3A等）:**
- https://www.silabs.com/developers/usb-to-uart-bridge-vcp-drivers

**CH340搭載ボード（多くの汎用ESP32ボード等）:**
- http://www.wch.cn/downloads/CH341SER_ZIP.html

### 手順

1. ESP32をUSBケーブルでPCに接続
2. ブロックエディタでプログラムを作成
3. 「**書き込み**」→「**USB**」を選択
4. ブラウザのシリアルポート選択ダイアログでESP32のポートを選択
5. 書き込み開始（約30秒）→ 完了後自動再起動

### トラブルシューティング

| 症状 | 対処 |
|------|------|
| ポートが表示されない | USBドライバーをインストール |
| 書き込み失敗 | BOOTボタンを押しながら書き込み開始 |
| タイムアウトエラー | 別のUSBポート、別のケーブルを試す |

## 方式2: WiFi OTA（オプション）

ケーブルなしで高速に書き込める方式です。初回のみセットアップが必要です。

### 前提条件

- **ファームウェア書き込み済み**（初回のみUSBで実施）
- WiFi設定済み（SSID/パスワード）
- DigiCode Finder（デスクトップアプリ）

### 手順

1. DigiCode Finderを起動、デバイスを自動検出
2. 対象デバイスの「選択」でIPアドレスをコピー
3. 「書き込み」→「WiFi OTA」→ IPアドレスをペースト
4. 書き込み開始（約15秒）

→ セットアップ手順の詳細: [OTA設定ガイド](./05-ota-guide.md)

## 方式3: BLE（オプション）

Bluetooth経由の書き込み方式。ケース入りデバイスや、WiFi環境がない場所での更新に便利です。

### 前提条件

- **ファームウェア書き込み済み**（初回のみUSBで実施）
- Web Bluetooth対応ブラウザ（Chrome、Edge）

### 対応ボード

BLE対応（supportsBle: true）のESP32系ボード全般。

### 手順

1. プログラムを作成
2. 「書き込み」→「BLE」を選択
3. 「デバイスを検索」をクリック、Bluetoothスキャン開始
4. 「DigiCode-XXXXXX」を選択してペアリング
5. 書き込み開始（約40秒）

→ セットアップ手順の詳細: [OTA設定ガイド](./05-ota-guide.md)

## ファームウェア初期書き込み（WiFi OTA / BLE使用時のみ）

WiFi OTA または BLE を使いたい場合に、最初の1回だけ実施します。

### 手順

1. 左メニューの「**ファームウェア書き込み**」をクリック
2. ESP32をUSBケーブルでPCに接続
3. 「INSTALL」ボタンをクリック
4. シリアルポートを選択
5. 書き込み完了を待つ（約1分）

### WiFi設定（WiFi OTA用）

ファームウェア書き込み後、WiFi OTAを使うには：

1. 「WiFi設定」をクリック
2. シリアルポートを選択して接続
3. SSIDとパスワードを入力
4. 「接続テスト」をクリック
5. 接続成功後、固定IPアドレスが表示される

---

## シリアルモニタで動作確認

プログラムのデバッグに便利です。

1. ESP32をUSB接続
2. サイドバーの「シリアルモニタ」をクリック
3. ポートを選択して「接続」
4. ボーレート: 115200

## 書き込み方式の選び方

| 状況 | 推奨方式 |
|------|---------|
| 初心者・通常の開発 | **USB**（最も確実） |
| WiFi OTAセットアップ済 | WiFi OTA（最速） |
| WiFi環境がない | USB または BLE |
| ケース入りデバイス | BLE または WiFi OTA |
| 教室での一括更新 | WiFi OTA |
| コンパイル枠を節約したい | ローカルコンパイルサーバー |
| トラブル復旧 | **USB**（最も確実） |
