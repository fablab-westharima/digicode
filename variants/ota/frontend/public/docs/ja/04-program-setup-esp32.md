# プログラム書き込み — ESP32 系

**最終更新:** 2026-05-02

ESP32 系ボードへのプログラム書き込み手順 (USB / WiFi OTA / BLE / ローカルコンパイル) を方式別にまとめます。

---

## 🚀 USB 書き込み — 5 ステップ

最も確実で初心者にも推奨。約 30 秒で完了。

1. **ESP32 を USB ケーブル (データ通信対応) で PC に接続**
2. ブロックエディタでプログラムを作成 (またはサンプル読み込み)
3. 右上「**書き込み**」ボタン → **「USB」** を選択
4. ブラウザのシリアルポート選択ダイアログで ESP32 のポートを選択
5. 書き込み完了後、ESP32 が自動再起動 ✅

> 💡 **ポートが出ない場合は USB ドライバ (CP2102 / CH340) を install + PC 再起動** ([詳細](#usb-ドライバ-install))。

## 4 つの書き込み方式 — クイック比較

| 方式 | 速度 | 事前準備 | 備考 |
|---|---|---|---|
| **🥇 USB 直接** | ~30 秒 | USB ドライバ | 最も確実 |
| **🥈 WiFi OTA** | ~15 秒 | FW + WiFi 設定 (初回 USB) | 最速、複数台同時 OK |
| **🥉 BLE** | ~40 秒 | FW (初回 USB) | WiFi 不要 |
| **⚡ ローカルコンパイル** | (組合せ用) | Docker | クラウド枠節約 |

## 方式 1: USB 直接書き込み (基本・推奨)

### USB ドライバ install

ESP32 が認識されない場合 (ポート選択ダイアログにデバイスが出ない場合) はドライバの install が必要です。

| チップ | 対象ボード例 | DL URL |
|---|---|---|
| **CP2102** | ESP32-DevKitC、M5StampS3A、多くの M5Stack 系 | https://www.silabs.com/developers/usb-to-uart-bridge-vcp-drivers |
| **CH340** | 多くの汎用 ESP32 ボード | http://www.wch.cn/downloads/CH341SER_ZIP.html |

install 後は **PC を再起動** してください。

### トラブルシューティング

| 症状 | 対処 |
|---|---|
| ポートが表示されない | USB ドライバを install + 再起動、データ通信対応ケーブル確認 |
| 書き込み失敗 | BOOT / BOOTSTRAP ボタンを押しながら書き込み開始 |
| タイムアウトエラー | 別の USB ポート / 別のケーブルを試す |

---

## 方式 2: WiFi OTA (オプション)

ケーブルなしで最速書き込み。複数台同時更新にも対応。

### 前提条件

- **ファームウェア書き込み済み** (初回のみ USB で実施、[詳細](#ファームウェア初期書き込み))
- WiFi 設定済み (SSID / パスワード)
- WiFi ルーター (本体と同 LAN)
- (Windows + Bonjour 未 install 環境のみ) [DigiCode Finder](https://github.com/fablab-westharima/DigiCode-Finder) — mDNS 代替手段。Mac / Linux は標準 mDNS 対応で不要

### 手順

1. ブラウザで DigiCode を開く (または DigiCode Finder で IP アドレス取得)
2. 「**書き込み**」 → 「**WiFi OTA**」 を選択
3. (mDNS 自動検出 OK の環境では device 一覧から選択 / それ以外は IP アドレスを入力)
4. 書き込み開始 (~15 秒)

> ⚠️ **Windows 上級者向け**: WiFi OTA 構築は前提条件が複数あり Windows 初学者には負担大。USB / BLE 推奨。詳細は [ローカルコンパイルサーバー § Windows](./local-compile-server.md#-まず-推奨度と前提知識)。

→ セットアップ詳細: [OTA 設定ガイド](./05-ota-guide.md)

## 方式 3: BLE (オプション)

Bluetooth 経由の書き込み。ケース入りデバイス、WiFi 環境がない場所で便利。

### 前提条件

- **ファームウェア書き込み済み** (初回のみ USB で実施)
- Web Bluetooth 対応ブラウザ (Chrome / Edge)
- 対応ボード: BLE 対応 ESP32 (`supportsBle: true`)

### 手順

1. ブロックエディタでプログラムを作成
2. 「**書き込み**」 → 「**BLE**」 を選択
3. 「**デバイスを検索**」 → Bluetooth スキャン開始
4. 「**DigiCode-XXXXXX**」 を選択してペアリング
5. 書き込み開始 (~40 秒)

→ セットアップ詳細: [OTA 設定ガイド](./05-ota-guide.md)

---

## 方式 4: ローカルコンパイルサーバー (高速化オプション)

自分の PC でコンパイルすることで、クラウドコンパイル枠を消費せずに高速ビルド。書き込み方式 (USB / WiFi OTA / BLE) と直交、組み合わせ可能。cache HIT で約 1 ms。

→ 詳細: [ローカルコンパイルサーバー](./local-compile-server.md)

## 詳細・概念

### 用語

| 用語 | 説明 |
|---|---|
| **プログラム書き込み** | ブロックエディタで作成したプログラムを書き込む操作 (毎回) |
| **ファームウェア書き込み** | WiFi OTA / BLE 用の基盤ソフトウェアを書き込む操作 (**WiFi OTA / BLE 使用時の初回のみ**) |

詳しくは [共通手順 § 用語](./01-program-setup-common.md#用語-プログラム書き込み-vs-ファームウェア書き込み) を参照。

### ファームウェア初期書き込み

WiFi OTA または BLE を使う場合、最初の 1 回だけ USB 経由で実施します。

1. 左メニューの **「ファームウェア書き込み」** をクリック
2. ESP32 を USB ケーブルで PC に接続
3. **「INSTALL」** ボタンをクリック
4. シリアルポートを選択
5. 書き込み完了を待つ (~1 分)

### WiFi 設定 (WiFi OTA 用)

ファームウェア書き込み後、WiFi OTA を使うには:

1. 「**WiFi 設定**」をクリック
2. シリアルポートを選択して接続
3. SSID とパスワードを入力
4. 「**接続テスト**」をクリック
5. 接続成功後、固定 IP アドレスが表示される

### シリアルモニタで動作確認

プログラムのデバッグに便利。

1. ESP32 を USB 接続
2. サイドバーの **「シリアルモニタ」** をクリック
3. ポートを選択して **「接続」**
4. ボーレート: **115200**

## 書き込み方式の選び方

| 状況 | 推奨方式 |
|---|---|
| 初心者・通常の開発 | **🥇 USB** (最も確実) |
| WiFi OTA セットアップ済 | 🥈 **WiFi OTA** (最速) |
| WiFi 環境がない | USB または BLE |
| ケース入りデバイス | 🥉 BLE または WiFi OTA |
| 教室での一括更新 | WiFi OTA |
| コンパイル枠を節約したい | ⚡ ローカルコンパイルサーバー |
| トラブル復旧 | **🥇 USB** (最も確実) |
