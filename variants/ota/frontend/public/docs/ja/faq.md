## アカウント・ログイン

### Q: アカウント登録は必要ですか？

**A:** いいえ、**ゲストのままでもプログラム作成・書き込みは可能**です。ただしプロジェクトをクラウドに保存したい場合はアカウント登録が必要です（Free プランから利用可能）。

### Q: パスワードを忘れました

**A:** ログイン画面から「パスワードを忘れた方」のリンクから、登録済みメールアドレスにリセットリンクを送信できます。

### Q: 複数のデバイスでログインできますか？

**A:** はい、同じアカウントで複数のデバイスからログインできます。プロジェクトはクラウドに保存されるため、どのデバイスからでもアクセスできます。

## エディタ・ブロック

### Q: ブロックが見つかりません

**A:** 以下を確認してください：

1. **ロボットモード** - 使用するモードを選択していますか？DigiCode は以下の 7 モードがあります：
   - Humanoid（2足歩行）
   - Wheel（車輪型）
   - Transform（変形ロボット）
   - Home Assistant（IoT 連携）
   - Generic（汎用）
   - All Blocks（すべてのブロック）
   - Custom（カスタム）

2. **カテゴリ** - ツールボックスの各カテゴリを展開して確認してください

3. **ボード対応** - 選択したボードによって一部ブロックは非表示になります
   - 例: BLE ブロックは Pico W では表示されません（BLE 非対応）

### Q: ブロックを接続できません

**A:** ブロックには接続できる形状が決まっています：

- **凹凸形状** - 上下に接続するブロック（実行順序）
- **丸い穴** - 値を入力するブロック（数値、文字列など）
- **三角形** - 条件を入力するブロック（真偽値）

形状が合わないブロック同士は接続できません。

### Q: ボードを切り替えるとブロックが消えました

**A:** ボードを切り替えると、対応していないブロックがワークスペースから削除されることがあります（例: Pico W で BLE ブロック）。切り替え前にプロジェクトを保存してください。

### Q: 日本語が文字化けします

**A:** シリアル通信で日本語を使用する場合、文字化けが発生することがあります。シリアルモニタのボーレートが一致しているか確認してください（通常 115200）。

## プログラム書き込み

### Q: 「プログラム書き込み」と「ファームウェア書き込み」の違いは？

**A:**

| 操作 | 内容 | 頻度 |
|------|------|------|
| **プログラム書き込み** | ブロックエディタで作ったプログラムを書き込む | 毎回（USB / WiFi OTA / BLE） |
| **ファームウェア書き込み** | WiFi OTA / BLE 用の基盤ソフトウェアを書き込む | **WiFi OTA / BLE 使用時の初回のみ** |

**USB 書き込みのみで運用する場合、ファームウェア書き込みは不要です。**

### Q: どの書き込み方式が推奨ですか？

**A:** 初心者・通常の開発には **USB 直接書き込み** が最も確実でおすすめです。慣れてきてケーブルなしで更新したくなったら、WiFi OTA / BLE のセットアップを行ってください。

### Q: コンパイルに失敗します

**A:** よくある原因と対処法：

| エラー | 原因 | 対処 |
|--------|------|------|
| ライブラリが見つからない | 必要なライブラリ未インストール | サーバー側で自動インストールされます。再試行 |
| 構文エラー | ブロックの接続ミス | 未接続のブロックがないか確認 |
| メモリ不足 | プログラムが大きすぎる | 不要なブロックを削除 |
| コンパイル枠超過 | プラン上限に到達 | プランをアップグレードまたはローカルコンパイルサーバーを使用 |

### Q: ESP32 が認識されません

**A:** 以下を順番に確認してください：

1. **USB ケーブル** - データ通信対応か（充電専用は不可）
2. **ドライバー** - CP2102 / CH340 ドライバーがインストールされているか
3. **ポート** - 別の USB ポートを試す
4. **ブラウザ** - Chrome / Edge を使用しているか（Web Serial API 対応）

### Q: 書き込み中にエラーが出ます

**A:** よくあるエラーと対処：

- **「A fatal error occurred: Failed to connect」**
  → ESP32 の BOOT ボタンを押しながら書き込みを開始
- **「Timed out waiting for packet header」**
  → ボーレートを下げて再試行
- **「No serial data received」**
  → USB ケーブルとポートを確認

### Q: コンパイルに時間がかかります

**A:** 初回は 30 秒〜1 分程度かかります。高速化したい場合は [ローカルコンパイルサーバー](./local-compile-server.md) をご利用ください（Pro プラン以上推奨）。

## WiFi OTA / BLE

### Q: デバイスが見つかりません（WiFi OTA）

**A:** 以下を確認してください：

1. **同じ WiFi ネットワーク** - PC と ESP32 が同じネットワークに接続されているか
2. **ファイアウォール** - mDNS（ポート 5353）がブロックされていないか
3. **Bonjour**（Windows のみ） - Bonjour Print Services がインストールされているか
4. **ESP32 の WiFi 接続** - シリアルモニタで IP アドレスを確認できるか

### Q: BLE でデバイスが見つかりません

**A:**

1. **ブラウザ対応** - Chrome / Edge を使用しているか（Web Bluetooth 対応）
2. **ボード対応** - ESP32-S2 は BLE 非対応、RP2040 Pico W も BLE 非対応
3. **macOS** - macOS のキャッシュによりデバイス名が古いまま表示される場合があります（OS レベルの問題）。Windows では正常動作します

### Q: OTA 更新が途中で止まります

**A:**

1. WiFi 信号が強い場所で実行
2. ESP32 を電源に近い場所に配置
3. 他の WiFi 機器を一時的に切断
4. ルーターを再起動

## ハードウェア

### Q: どのESP32ボードを使えばいいですか？

**A:** DigiCode は ESP32 系専用です。以下のボードで動作確認済みです：

- **ESP32-DevKitC** - 最も一般的
- **M5StampS3A** - DigiCode 推奨ボード（専用ブレイクアウトボード開発中）
- **ESP32-S3** - 高性能
- **XIAO ESP32S3** - カメラ対応

詳細は [推奨ハードウェア](./recommended-hardware.md) を参照してください。

### Q: サーボが動きません

**A:**

1. **電源** - サーボの消費電流は大きいため、USB 給電では不足することがあります。外部電源（5V/2A 以上）を使用
2. **ピン番号** - ピン設定が正しいか確認
3. **初期化** - サーボ接続ブロックを配置しているか

### Q: センサーの値がおかしいです

**A:**

1. **配線** - VCC / GND / 信号線が正しく接続されているか
2. **電圧** - センサーの動作電圧（3.3V / 5V）を確認
3. **アナログピン** - ADC 対応ピン（GPIO32-39）を使用しているか

### Q: Humanoid ロボットがうまく歩きません

**A:**

1. **キャリブレーション** - ホームポジションブロックで直立姿勢を確認
2. **サーボパルス幅** - ピン設定ページでパルス幅を調整（500-2400μs）
3. **速度** - 速度を遅くして安定性を確認
4. **重心** - 電池の位置を調整

## プロジェクト管理

### Q: プロジェクトを保存できません

**A:**

1. ログインしているか（ゲストはローカル保存のみ）
2. プロジェクト名を入力しているか
3. ネットワーク接続は正常か

### Q: プロジェクトが消えました

**A:**

1. 正しいアカウントでログインしているか
2. プロジェクト一覧で検索
3. 誤って削除した場合、復元はできません

### Q: プロジェクトを共有できますか？

**A:** クラス機能を利用すれば、教員が生徒に課題として配布・生徒が答案を提出という形での共有が可能です（Enterprise プラン）。一般公開のプロジェクト共有機能は開発予定です。

### Q: サンプルプロジェクトを編集できますか？

**A:** サンプルプロジェクトを読み込むと、自動的にコピーが作成されます。元のサンプルは変更されません。

## クラス機能

### Q: クラス機能とは？

**A:** 学校・教育機関向けのクラス管理機能です（**Enterprise プラン**）。

| 役割 | できること |
|------|-----------|
| **教員** | クラス作成・課題配布・答案確認・クラス複製 |
| **生徒** | クラス参加（招待 URL）・課題提出・提出履歴確認 |

### Q: 生徒はアカウント登録が必要ですか？

**A:** はい、生徒も DigiCode のアカウントが必要です。教員から送られた招待 URL 経由でクラスに参加すると、Enterprise プラン相当の権限が付与されます（ファブラボ西播磨から付与される形）。

### Q: クラス機能はどのプランで使えますか？

**A:** **Enterprise プラン** 専用です。個人の Free / Lite / Pro プランでは利用できません。

### Q: 課題の Blockly XML はどこに保存されますか？

**A:** 課題・答案の Blockly XML は HPE ML30 上のクラスサーバー（`class.digital-fab.jp`）に保存されます。D1 には基本的な管理情報（クラス ID、メンバー等）のみ保存されます。

### Q: クラスサーバーがダウンしたらどうなりますか？

**A:** クラス機能（課題配布・答案提出）は利用できなくなりますが、DigiCode の通常のプログラム作成・書き込みは継続可能です。影響を課題・答案操作だけに限定する設計になっています。

## プラン・料金

### Q: 無料で使えますか？

**A:** はい、**Free プラン**で基本機能を無料で使用できます。

| プラン | 向いている方 | 月額 | クラウドコンパイル | クラス機能 |
|--------|------------|:----:|:----------------:|:--------:|
| **Free** | お試し・ローカルコンパイル向け | ¥0 | 月 50 回 | — |
| **Lite** | 個人ホビイスト向け | — | 月 250 回 | — |
| **Pro** | 開発者・Maker 向け | — | 月 500 回 | — |
| **Enterprise** | 教育機関・チーム向け | — | 無制限 | ○ |

最新の価格は [プランページ](/plan) を参照してください。

### Q: プランを変更するには？

**A:** 右上のアカウントメニュー → 「プラン・お支払い」から変更できます。Stripe のカスタマーポータルで請求情報・プラン変更・解約が可能です。

### Q: 招待アカウントとは？

**A:** ファブラボ西播磨から Enterprise 相当の権限を付与されたアカウントです。自分でプランを契約する必要はありません。ただし自分で上位プランを契約することも可能（その場合は招待による権限付与は解除されます）。

### Q: 教育機関向けライセンスはありますか？

**A:** **Enterprise プラン** で教育機関向けのクラス機能を提供しています。まとまった数のライセンスや特別対応が必要な場合は、[ファブラボ西播磨](https://fablab-westharima.com/) までお問い合わせください。

### Q: ローカルコンパイルサーバーを使うとクラウド枠を消費しますか？

**A:** いいえ、ローカルコンパイルサーバーを使用するとクラウドコンパイル枠を消費しません。Pro / Enterprise プランでの高速化・無制限利用に最適です。→ [ローカルコンパイルサーバー](./local-compile-server.md)

## その他

### Q: 対応ブラウザは？

**A:**

| ブラウザ | 対応状況 | 備考 |
|---------|---------|------|
| Chrome | ○ 推奨 | Web Serial / Web Bluetooth 対応 |
| Edge | ○ | Web Serial / Web Bluetooth 対応 |
| Firefox | △ | USB / BLE 書き込み非対応 |
| Safari | △ | USB / BLE 書き込み非対応 |

USB / BLE 書き込みを使用する場合は、Chrome または Edge を使用してください。

### Q: スマートフォンで使えますか？

**A:** ブロックエディタはスマートフォンでも表示できますが、USB 書き込みは非対応です。WiFi OTA 書き込みであればスマートフォンからでも可能です。

### Q: オフラインで使えますか？

**A:** 現在はオンライン接続が必要です。ローカルコンパイルサーバーを使えば、コンパイル自体はオフラインで動作します。

### Q: バグを見つけました

**A:** [GitHub Issues](https://github.com/fablab-westharima/DigiCode/issues) でご報告ください。以下の情報を含めていただけると助かります：

- ブラウザとバージョン
- OS
- 使用しているプラン
- 再現手順
- エラーメッセージ（あれば）
