## 基本ブロック

ESP32 の入出力・プログラム制御の基礎となるブロックです。

### プログラム構造

| ブロック名 | 説明 | 備考 |
|-----------|------|------|
| **セットアップ** | プログラム起動時に 1 回だけ実行 | 初期化処理を配置 |
| **ループ（繰り返し）** | 繰り返し実行されるメイン処理 | センサー読み取りや制御処理を配置 |

### デジタル入出力

| ブロック名 | 説明 | パラメータ |
|-----------|------|-----------|
| **ピンモード設定** | ピンの入出力モードを設定 | ピン番号、モード(INPUT/OUTPUT/INPUT_PULLUP) |
| **デジタル出力** | ピンに HIGH/LOW を出力 | ピン番号、値(HIGH/LOW) |
| **デジタル入力** | ピンの状態を読み取る | ピン番号 → true/false |
| **内蔵LED ON** | ESP32 内蔵 LED(GPIO2) を点灯 | — |
| **内蔵LED OFF** | ESP32 内蔵 LED(GPIO2) を消灯 | — |

### アナログ入出力

| ブロック名 | 説明 | パラメータ |
|-----------|------|-----------|
| **アナログ入力** | アナログ値を読み取る | ピン番号(GPIO32-39 推奨) → 0-4095 |
| **PWM出力** | PWM 信号を出力 | ピン番号、デューティ(0-255) |

### 時間・タイミング

| ブロック名 | 説明 | パラメータ |
|-----------|------|-----------|
| **待つ** | 指定時間待機 | 時間(ミリ秒) |
| **経過時間** | 起動からの経過時間(ms)を取得 | → ミリ秒 |

### シリアル通信

| ブロック名 | 説明 | パラメータ |
|-----------|------|-----------|
| **シリアル通信開始** | シリアル通信を初期化 | ボーレート(9600/57600/115200) |
| **シリアル出力** | シリアルモニタに出力 | 出力値 |
| **シリアル出力(改行あり)** | 改行付きでシリアル出力 | 出力値 |

### 数値変換

| ブロック名 | 説明 | パラメータ |
|-----------|------|-----------|
| **数値マッピング** | 数値を別の範囲に変換 | 値、入力Min/Max、出力Min/Max → 変換値 |

### ESP32 ユーティリティ

| ブロック名 | 説明 | パラメータ |
|-----------|------|-----------|
| **ESP32を再起動** | ESP32 をリセット | — |
| **ディープスリープ** | 省電力ディープスリープ | 秒数（復帰後は最初から再実行） |
| **ライトスリープ** | ライトスリープ | 秒数（復帰後に続きを実行） |
| **空きメモリ（バイト）** | 空きヒープメモリ取得 | → バイト数 |
| **ESPチップID** | チップのユニーク ID | → 文字列 |
| **CPU周波数（MHz）** | CPU 動作周波数を取得 | → MHz |
| **音を鳴らす** | 指定周波数の音を出力 | ピン番号、周波数(Hz)、時間(ms) |

## 割り込み・時刻

ピン割り込み・定期タイマー・時刻取得のブロックです。

> **注意:** 割り込みハンドラ内で `delay()` や `Serial.print()` を呼ぶと ESP32 がクラッシュします。フラグ変数は `volatile` で宣言し、ループ側で処理してください。

### ピン割り込み

| ブロック名 | 説明 | パラメータ |
|-----------|------|-----------|
| **割り込み設定** | ピン変化時にハンドラを実行 | ピン、モード(RISING/FALLING/CHANGE)、ハンドラ |
| **割り込み解除** | 割り込みハンドラを解除 | ピン |
| **割り込み確認** | 割り込みフラグ確認・実行 | ループ内に配置 |

### 定期タイマー（Ticker）

**対応:** ESP32 専用（Ticker.h 使用）

| ブロック名 | 説明 | パラメータ |
|-----------|------|-----------|
| **タイマー開始** | 指定間隔でハンドラを定期実行 | 間隔(ms)、ハンドラ |
| **タイマー停止** | 定期タイマーを停止 | — |
| **タイマー確認** | タイマーフラグ確認・実行 | ループ内に配置 |

### NTP 時刻同期

**対応:** WiFi 対応ボード（WiFi接続が必要）

| ブロック名 | 説明 | パラメータ |
|-----------|------|-----------|
| **NTP同期** | NTP サーバーで時刻同期 | サーバー URL、タイムゾーン(秒、JST=32400) |
| **Unix時刻取得** | Unix タイムスタンプを取得 | → 秒数 |
| **フォーマット済み時刻** | 書式指定で時刻文字列取得 | フォーマット(%Y-%m-%d %H:%M:%S 等) → 文字列 |
| **時刻コンポーネント取得** | 年/月/日/時/分/秒を個別取得 | 項目選択 → 数値 |

### RTC（リアルタイムクロック）

**ライブラリ:** RTClib（Dockerfile 内蔵）

| ブロック名 | 説明 | パラメータ |
|-----------|------|-----------|
| **RTC初期化** | DS3231/DS1307 を I2C で初期化 | モデル選択 |
| **RTC時刻設定** | RTC に日時を設定 | 年/月/日/時/分/秒 |
| **RTC時刻取得** | 年/月/日/時/分/秒/Unix時刻 | 項目選択 → 数値 |
| **RTCフォーマット済み時刻** | "YYYY-MM-DD HH:MM:SS" 形式で取得 | → 文字列 |

## データ（配列）

配列を操作するブロックです。

| ブロック名 | 説明 | パラメータ |
|-----------|------|-----------|
| **配列を作成** | 1/2/3 次元配列を作成 | 変数名、型、次元、サイズ |
| **配列要素設定** | 配列の要素に値を設定 | 変数名、インデックス、値 |
| **配列要素取得** | 配列の要素を取得 | 変数名、インデックス → 値 |
| **配列サイズ取得** | 配列の要素数を取得 | 変数名 → 数値 |
| **配列内容** | 配列の初期値を定義 | 要素数、各要素の値 |

## 対応ボード

DigiCode は ESP32 系専用ブロックエディタです。ボードに応じてブロックの表示/非表示が自動的に切り替わります。

| フラグ | 意味 | 対応ボード例 |
|--------|------|------------|
| `supportsWifi` | WiFi ブロック・HTTP・MQTT・HA が利用可能 | ESP32 系全般、Pico W、Nano RP2040 Connect |
| `supportsOta` | WiFi OTA 書き込み（DigiCode OTA ファームウェア）が利用可能 | ESP32 系全般（RP2040 は非対応） |
| `supportsBle` | BLE ブロック（NimBLE）が利用可能 | ESP32 系全般（RP2040 は非対応） |

> **注意:** Pico W は WiFi に対応していますが BLE には対応していません。RFID/CAN/Camera ブロックは ESP32 専用です。
