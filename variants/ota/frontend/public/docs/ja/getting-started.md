# はじめかた

**最終更新:** 2026-05-02

DigiCode は ESP32 マイコンを使ったロボット・IoT デバイスのプログラミングを、ブロックで簡単に行えるビジュアル開発環境です。本ガイドでは **USB ケーブルで「LED 点滅」を動かすまで** を最短手順で説明します。

---

## 🚀 5 分で動かす — 最短フロー

1. **USB ドライバを install** (CP2102 または CH340、未 install の場合のみ、後述)
2. **DigiCode を開く** → 左上「**サンプル**」 → **「LED 点滅」** を選択
3. **ESP32 を USB ケーブルで PC に接続**
4. 右上「**書き込み**」ボタン → **「USB」** を選択 → ブラウザで ESP32 のポートを選択
5. **書き込み完了後、ESP32 内蔵 LED (GPIO2) が 1 秒間隔で点滅** ✅

完了です。これで DigiCode が動く環境が整いました。

> 💡 **ゲスト利用 OK**: アカウント作成は任意。プログラムをクラウドに保存したい場合のみ登録してください ([詳細](#ゲスト利用とアカウント))。

---

## 🟡 始める前に確認

| 項目 | 内容 |
|---|---|
| **ハードウェア** | ESP32 開発ボード ([推奨リスト](./recommended-hardware.md)) + USB ケーブル (Type-C または Micro-USB、**データ通信対応** — 充電専用ケーブルは NG) |
| **ソフトウェア** | Web ブラウザ (Chrome / Edge 推奨、Web Serial API 対応) |
| **ドライバ** | USB-シリアル変換チップに応じて CP2102 または CH340 (下記表) |

## 5 ステップで完了

### ステップ 1: USB ドライバを install

ESP32 を PC に接続する前に、USB-シリアル変換ドライバが必要です。

| チップ | 対象ボード例 | ダウンロード |
|---|---|---|
| **CP2102** | ESP32-DevKitC、M5StampS3A 等 | https://www.silabs.com/developers/usb-to-uart-bridge-vcp-drivers |
| **CH340** | 多くの汎用 ESP32 ボード | http://www.wch.cn/downloads/CH341SER_ZIP.html |

ボードに使われているチップは商品説明を確認してください。インストール後、PC を再起動すると認識されます。

> 💡 **M5Stack 系の多くは CP2102**。汎用の安価なボードは CH340 が多いです。

### ステップ 2: DigiCode を開く

[DigiCode](https://digital-fab.jp) にアクセスしてエディタ画面を開きます。**ゲストのまま使えます** (登録不要)。

### ステップ 3: サンプル「LED 点滅」を読み込む

1. 左上の **「サンプル」** ボタンをクリック
2. **「基本」** カテゴリ → **「LED 点滅」** を選択
3. ワークスペースに自動でブロックが配置されます

> ✨ **成功体験 first**: まずサンプルで「動く」ことを確認してから、ブロックの仕組みを理解する方が定着率が高いです。自分でブロックを並べたい方は [後述](#自分でブロックを置く参考) を参照。

### ステップ 4: ESP32 を USB 接続して書き込む

1. ESP32 を USB ケーブルで PC に接続
2. エディタ右上の **「書き込み」** ボタンをクリック
3. **「USB」** を選択
4. ブラウザのシリアルポート選択ダイアログで ESP32 のポートを選択 (例: `COM3`、`/dev/ttyUSB0`、`/dev/cu.usbserial-XXXX`)
5. 書き込み開始 → 完了後 ESP32 が自動再起動

### ステップ 5: 動作確認

ESP32 内蔵 LED (GPIO2) が **1 秒間隔で点滅** すれば成功です。

## トラブル: ポートが表示されない

### 「USB」を選択してもポート選択ダイアログにデバイスが出ない

最も多い原因は USB ドライバ未 install です:

1. **ステップ 1 のドライバ** を install していない / 再起動していない → install + 再起動
2. **データ通信対応の USB ケーブル** か確認 (充電専用ケーブルは ESP32 が PC に認識されません)
3. ESP32 を抜き差し → ブラウザで「書き込み」ボタンを再クリック

それでも解決しない場合は [トラブルシューティング](./troubleshooting.md) を参照。

## 詳細・概念

### プログラム書き込みとファームウェア書き込みの違い

| 操作 | 内容 | タイミング |
|---|---|---|
| **プログラム書き込み** | ブロックエディタで作ったプログラムを ESP32 に書き込む | 毎回 (USB / WiFi OTA / BLE) |
| **ファームウェア書き込み** | WiFi OTA / BLE 用の基盤ソフトウェアを書き込む | **WiFi OTA / BLE を使いたい場合の初回のみ** |

**USB のみで運用する場合、ファームウェア書き込みは不要** です。

### ゲスト利用とアカウント

- **ゲスト**: プログラム作成・書き込み・サンプル利用すべて可能。プログラム保存はファイルダウンロードのみ
- **アカウント (Free)**: クラウドにプログラム保存可能、AI ブロック生成・要望投稿などの追加機能 → [プラン一覧](./index.md#プラン)

登録は **「ログイン」 → 「新規登録」** からメールアドレスとパスワードを入力するだけです。

### 自分でブロックを置く (参考)

サンプルではなく自分でブロックを組みたい方向け。LED 点滅の最小コードは以下:

**セットアップ:**
```
ピンモード設定 [2] を [出力]
```

**ループ:**
```
ずっと繰り返す:
  デジタル書き込み [2] を [HIGH]
  待機 [1000] ミリ秒
  デジタル書き込み [2] を [LOW]
  待機 [1000] ミリ秒
```

GPIO2 が ESP32 の内蔵 LED に対応しています。詳細なブロックの使い方は [ブロックリファレンス](./block-reference.md) を参照。

### サンプルカテゴリ

| カテゴリ | 内容 |
|---|---|
| **基本** | LED 点滅、シリアル通信、ボタン入力 |
| **センサー** | 超音波、温湿度、光、加速度 |
| **モーター** | サーボ、DC モーター、ステッピング |
| **ロボット** | ヒューマノイド、車輪型、変形 |
| **IoT** | WiFi、MQTT、Home Assistant |
| **競技** | ライントレース、マイクロマウス |

## 次に試す

### ケーブルなしで書き込む (中級者以上推奨)

毎回 USB ケーブルを接続するのが不便な場合は、無線書き込みを設定できます。

- **WiFi OTA** — WiFi 経由で最速書き込み (複数台同時更新にも対応)
- **BLE** — Bluetooth 経由 (WiFi 環境不要、ケース入りデバイスに便利)

初回のみ USB で「**ファームウェア書き込み**」+ WiFi 設定が必要です (約 5 分)。

> 📘 **詳細**: [OTA 設定ガイド](./05-ota-guide.md)

### クラス機能 (Enterprise プラン)

- **教員**: クラスを作成し、生徒に招待 URL を共有
- **生徒**: 招待 URL からクラスに参加し、課題に取り組む

→ 詳細: [FAQ (クラス機能)](./faq.md)

### 高頻度コンパイル (上級者向け)

クラウドコンパイル枠を超えて利用したい場合は、自分の PC にローカルコンパイルサーバーを構築できます。

→ 詳細: [ローカルコンパイルサーバー](./local-compile-server.md)
