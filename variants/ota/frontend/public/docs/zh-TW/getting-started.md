# 開始使用 - DigiCode

**最後更新:** 2025-12-28

DigiCode 是一個視覺化開發環境，讓您可以使用積木方塊輕鬆為 ESP32 微控制器的機器人和 IoT 裝置進行程式設計。

![設定4步驟](/docs/images/getting-started-steps.svg)

## 準備事項

### 硬體
- **ESP32 開發板** - 請參考[推薦硬體清單](./recommended-hardware.md)
- USB 線（Type-C 或 Micro-USB，視開發板而定）
- 感測器和馬達（依據您想製作的專案）

> **重要:** 為確保穩定運作，建議從[推薦硬體清單](./recommended-hardware.md)中列出的商店購買。

### 軟體
- 最新的網頁瀏覽器（建議 Chrome、Edge）
- 網路連線
- **DigiCode Finder**（WiFi OTA 燒錄用桌面應用程式，選用）

## 步驟1: 建立帳號並登入

1. 前往 DigiCode
2. 點選「註冊」
3. 輸入電子郵件和密碼
4. 註冊完成後自動登入

## 步驟2: 將韌體燒錄到 ESP32

使用 DigiCode 前，需要先將專用韌體燒錄到 ESP32。

1. 點選左側選單的「**韌體燒錄**」
2. 使用 USB 線將 ESP32 連接到電腦
3. 點選「INSTALL」按鈕
4. 選擇要連接的序列埠
5. 等待燒錄完成（約1分鐘）

> **注意:** 韌體只需燒錄一次，之後通常不需要重新燒錄。程式可以重複燒錄多次。

## 步驟3: 第一個程式（LED 閃爍）

### 建立程式

1. 顯示編輯器畫面
2. 從左側工具箱選擇以下積木：

#### 設定積木
```
設定:
  設定腳位模式 [2] 為 [輸出]
```

#### 迴圈積木
```
重複執行:
  數位寫入 [2] 為 [HIGH]
  等待 [1000] 毫秒
  數位寫入 [2] 為 [LOW]
  等待 [1000] 毫秒
```

### 燒錄到 ESP32

1. 點選「**燒錄**」按鈕
2. 選擇燒錄方式：

| 方式 | 用途 | 備註 |
|------|------|------|
| **WiFi OTA** | 一般更新（最快） | 使用 DigiCode Finder 偵測裝置 |
| **BLE** | 無線更新 | 需要支援 Web Bluetooth 的瀏覽器 |
| **USB** | 初次設定、穩定燒錄 | 需要連接傳輸線 |

#### WiFi OTA 燒錄（推薦）

1. 啟動 **DigiCode Finder**（桌面應用程式）
2. 會自動偵測網路上的 DigiCode 裝置
3. 點選「選擇」按鈕複製 IP 位址
4. 回到 DigiCode（瀏覽器）選擇「WiFi OTA」
5. 貼上 IP 位址並開始燒錄

> **下載 DigiCode Finder:**
> https://github.com/fablab-westharima/DigiCode-Finder/releases

#### USB 燒錄

1. 選擇「USB」
2. 選擇序列埠
3. 等待燒錄完成

### 確認運作

ESP32 的內建 LED（GPIO2）會以1秒間隔閃爍！

## 步驟4: 嘗試範例專案

1. 在編輯器中點選「範例」按鈕
2. 從類別中選擇：
   - **基礎** - LED 閃爍、序列通訊
   - **感測器** - 超音波感測器、溫濕度感測器
   - **競賽** - 循跡、迷宮老鼠
3. 選擇範例並載入
4. 根據需要修改腳位編號
5. 燒錄並確認運作

## 三種燒錄方式比較

| 方式 | 優點 | 缺點 |
|------|------|------|
| **WiFi OTA** | 快速、不需傳輸線 | 首次需燒錄韌體 |
| **BLE** | 不需傳輸線、可在 AP 限制環境使用 | 燒錄速度較慢 |
| **USB** | 穩定、任何環境都能使用 | 需要連接傳輸線 |

**推薦:** 首次用 USB 燒錄韌體，之後用 WiFi OTA 更新
