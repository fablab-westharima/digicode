# 開始使用

**最後更新:** 2026-04-21

DigiCode 是一款使用積木輕鬆為 ESP32 機器人和 IoT 裝置進行程式設計的視覺化開發環境。

本指南說明**使用 USB 線的基本流程**。如需 WiFi OTA / BLE 設定，請參閱 [OTA 設定指南](./05-ota-guide.md)。

---

## 所需物品

### 硬體

- **ESP32 開發板** — 請參閱 [推薦硬體清單](./recommended-hardware.md)
- **USB 線**（Type-C 或 Micro-USB，支援資料傳輸）
- 感測器和馬達（依據您要製作的項目）

### 軟體

- **網頁瀏覽器**（推薦 Chrome、Edge）
- **USB 驅動程式**（CP2102 或 CH340，詳見下方）

---

## 建立帳號（選用）

**訪客也可以建立和燒錄程式。** 如果想將程式儲存到雲端，請建立帳號。

1. 前往 DigiCode
2. 點擊「登入」→「新增帳號」
3. 輸入電子郵件和密碼

> **關於方案:** 先以訪客（Free）試用看看。→ [方案一覽](./index.md#方案)

---

## 步驟 1: 安裝 USB 驅動程式

連接 ESP32 到電腦前，需要安裝 USB 驅動程式。

| 晶片 | 開發板範例 | 下載 |
|------|----------|------|
| **CP2102** | ESP32-DevKitC、M5StampS3A 等 | https://www.silabs.com/developers/usb-to-uart-bridge-vcp-drivers |
| **CH340** | 許多通用 ESP32 開發板 | http://www.wch.cn/downloads/CH341SER_ZIP.html |

> 請查看開發板的產品說明確認使用的晶片。安裝後重新啟動電腦。

---

## 步驟 2: 建立程式（LED 閃爍）

1. 開啟 DigiCode 編輯器
2. 從工具箱放置以下積木：

**設定積木:**
```
設定:
  設定腳位模式 [2] 為 [輸出]
```

**迴圈積木:**
```
持續重複:
  數位寫入 [2] HIGH
  等待 [1000] 毫秒
  數位寫入 [2] LOW
  等待 [1000] 毫秒
```

> GPIO2 對應 ESP32 內建 LED。

---

## 步驟 3: 透過 USB 燒錄

1. **透過 USB 線連接 ESP32 到電腦**
2. 點擊編輯器的「**燒錄**」按鈕
3. 選擇「**USB**」
4. 在瀏覽器的序列埠選擇對話框中選擇 ESP32 的埠（例如 COM3、/dev/ttyUSB0）
5. 燒錄開始 → 完成後 ESP32 自動重新啟動

**確認:** ESP32 內建 LED（GPIO2）以 1 秒間隔閃爍。

### 如果埠未顯示

可能是 USB 驅動程式未正確安裝。回到步驟 1 安裝驅動程式。

---

## 步驟 4: 嘗試範例專案

1. 點擊編輯器的「**範例**」按鈕
2. 從類別選擇：
   - **基本** — LED 閃爍、序列通訊
   - **感測器** — 超音波感測器、溫濕度感測器
   - **競賽** — 循跡、微型鼠
3. 讀取範例
4. 依需求調整腳位編號
5. 透過 USB 燒錄並確認動作

---

## 程式燒錄和韌體燒錄的差異

| 操作 | 內容 | 時機 |
|------|------|------|
| **程式燒錄** | 燒錄用積木製作的程式 | 每次（USB / WiFi OTA / BLE） |
| **韌體燒錄** | 燒錄 WiFi OTA / BLE 用的基礎軟體 | **僅在使用 WiFi OTA / BLE 的第一次** |

**如果只使用 USB，不需要韌體燒錄。**

---

## 選用: 不用連接線燒錄

如果每次連接 USB 線感到不便，可以設定 WiFi OTA / BLE。

- **WiFi OTA** — 最快速的無線燒錄（支援批次更新）
- **BLE** — 藍牙傳輸（不需 WiFi，適合放入外殼的裝置）

需要一次性的 USB 設定：**韌體燒錄** + WiFi 設定（約 5 分鐘）。

> **建議中階以上使用:** WiFi OTA 需要網路設定，發生問題時的排查可能較複雜。建議先習慣 USB 燒錄後再切換。

→ 詳細: [OTA 設定指南](./05-ota-guide.md)
