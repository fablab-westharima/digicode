# 開始使用

**最後更新:** 2026-05-02

DigiCode 是一款使用積木輕鬆為 ESP32 機器人和 IoT 裝置進行程式設計的視覺化開發環境。本指南以最短路徑說明 **使用 USB 線讓「LED 閃爍」運作** 的步驟。

---

## 🚀 5 分鐘運行 — 最短流程

1. **安裝 USB 驅動程式** (CP2102 或 CH340,僅在未安裝時需要,詳見下方)
2. **開啟 DigiCode** → 左上「**範例**」 → 選擇 **「LED 閃爍」**
3. **透過 USB 線將 ESP32 連接至電腦**
4. 右上「**燒錄**」按鈕 → 選擇 **「USB」** → 在瀏覽器對話框中選擇 ESP32 的埠
5. **燒錄完成後,ESP32 內建 LED (GPIO2) 以 1 秒間隔閃爍** ✅

完成。DigiCode 環境已就緒。

> 💡 **訪客也可以使用**: 帳號建立為選用。僅在想將程式儲存到雲端時才需註冊 ([詳細](#訪客使用與帳號))。

## 🟡 開始前確認

| 項目 | 內容 |
|---|---|
| **硬體** | ESP32 開發板 ([推薦清單](./recommended-hardware.md)) + USB 線 (Type-C 或 Micro-USB,**支援資料傳輸** — 僅充電線無法使用) |
| **軟體** | 網頁瀏覽器 (推薦 Chrome / Edge — 需支援 Web Serial API) |
| **驅動程式** | 依開發板的 USB-序列轉換晶片選擇 CP2102 或 CH340 (下表) |

---

## 5 步驟完成

### 步驟 1: 安裝 USB 驅動程式

連接 ESP32 到電腦前,需先安裝 USB-序列轉換驅動程式。

| 晶片 | 開發板範例 | 下載 |
|---|---|---|
| **CP2102** | ESP32-DevKitC、M5StampS3A 等 | https://www.silabs.com/developers/usb-to-uart-bridge-vcp-drivers |
| **CH340** | 許多通用 ESP32 開發板 | http://www.wch.cn/downloads/CH341SER_ZIP.html |

請查看開發板的產品說明確認晶片類型。安裝後重新啟動電腦,讓 OS 認識新驅動程式。

> 💡 **多數 M5Stack 系列使用 CP2102**。通用平價 ESP32 開發板多使用 CH340。

### 步驟 2: 開啟 DigiCode

前往 [DigiCode](https://digital-fab.jp) 開啟編輯器。**訪客也可以使用** — 不需註冊。

### 步驟 3: 讀取「LED 閃爍」範例

1. 點擊左上的 **「範例」** 按鈕
2. 選擇 **「基本」** 類別 → **「LED 閃爍」**
3. 積木會自動配置到工作區

> ✨ **成功體驗優先**: 先以範例確認「能動」再學習積木機制,定著率較高。想自己擺放積木的人請參閱 [後述](#自行擺放積木參考)。

### 步驟 4: 透過 USB 連接 ESP32 並燒錄

1. 透過 USB 線將 ESP32 連接到電腦
2. 點擊編輯器右上的 **「燒錄」** 按鈕
3. 選擇 **「USB」**
4. 在瀏覽器的序列埠選擇對話框中選擇 ESP32 的埠 (例如 `COM3`、`/dev/ttyUSB0`、`/dev/cu.usbserial-XXXX`)
5. 燒錄開始 → 完成後 ESP32 自動重新啟動

### 步驟 5: 確認動作

ESP32 內建 LED (GPIO2) 以 **1 秒間隔閃爍** 即成功。

## 故障: 埠未顯示

### 在瀏覽器的埠對話框中沒有裝置出現

最常見原因是 USB 驅動程式未安裝:

1. **步驟 1 的驅動程式** 未安裝 / 安裝後未重新啟動 → 安裝 + 重新啟動
2. 確認 **USB 線支援資料傳輸** (僅充電線無法讓 ESP32 被電腦識別)
3. 拔除 ESP32 再插入 → 重新點擊「燒錄」

仍無法解決請參閱 [故障排除](./troubleshooting.md)。

## 詳細・概念

### 程式燒錄與韌體燒錄的差異

| 操作 | 內容 | 時機 |
|---|---|---|
| **程式燒錄** | 將積木編輯器製作的程式燒錄到 ESP32 | 每次 (USB / WiFi OTA / BLE) |
| **韌體燒錄** | 燒錄 WiFi OTA / BLE 用的基礎軟體 | **僅在使用 WiFi OTA / BLE 的第一次** |

**若僅使用 USB,不需要韌體燒錄。**

### 訪客使用與帳號

- **訪客**: 程式建立、燒錄、範例使用皆可。程式儲存僅限檔案下載
- **帳號 (Free)**: 可將程式儲存到雲端,並可使用 AI 積木產生、需求提交等附加功能 → [方案一覽](./index.md#方案)

註冊請從 **「登入」 → 「新增帳號」** 輸入電子郵件和密碼即可。

### 自行擺放積木 (參考)

不使用範例而想自行組合積木的人。LED 閃爍的最小程式碼如下:

**設定:**
```
設定腳位模式 [2] 為 [輸出]
```

**迴圈:**
```
持續重複:
  數位寫入 [2] HIGH
  等待 [1000] 毫秒
  數位寫入 [2] LOW
  等待 [1000] 毫秒
```

GPIO2 對應 ESP32 內建 LED。詳細的積木使用方法請參閱 [積木參考](./block-reference.md)。

### 範例類別

| 類別 | 內容 |
|---|---|
| **基本** | LED 閃爍、序列通訊、按鈕輸入 |
| **感測器** | 超音波、溫濕度、光、加速度 |
| **馬達** | 伺服、直流馬達、步進 |
| **機器人** | 人型、輪式、變形 |
| **IoT** | WiFi、MQTT、Home Assistant |
| **競賽** | 循跡、微型鼠 |

## 接下來嘗試

### 不用線燒錄 (建議中階以上)

如果每次接 USB 線感到不便,可以設定無線燒錄:

- **WiFi OTA** — 最快的無線燒錄 (支援多台同時更新)
- **BLE** — 藍牙傳輸 (不需 WiFi,適合放入外殼的裝置)

需要一次性的 USB 設定: **韌體燒錄** + WiFi 設定 (約 5 分鐘)。

> 📘 **詳細**: [OTA 設定指南](./05-ota-guide.md)

### 班級功能 (Enterprise 方案)

- **教師**: 建立班級並將邀請 URL 分享給學生
- **學生**: 透過邀請 URL 加入班級,進行作業

→ 詳細: [FAQ (班級功能)](./faq.md)

### 高頻率編譯 (進階使用者)

若要超出雲端編譯額度使用,可在自己的電腦上構建本地編譯伺服器。

→ 詳細: [本地編譯伺服器](./local-compile-server.md)
