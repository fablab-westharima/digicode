# 程式燒錄_ESP32 系列

**最後更新:** 2025-12-28

---

## 支援的開發板

- ESP32
- ESP32-S2 / S3
- ESP32-C2 / C3 / C5 / C6
- ESP8266
- 其他 ESP-IDF 對應開發板

---

## ESP32 的特性

ESP32 是搭載 WiFi/藍牙的高功能微控制器。

- **語言:** Arduino C++
- **連接方式:** WiFi OTA、BLE、USB
- **Bootloader:** 固定在 ROM 中（通常不需要更換）

---

## 術語定義

| 術語 | 說明 |
|------|------|
| **韌體** | DigiCode 的基礎程式。僅首次透過 USB 燒錄 |
| **程式** | 在積木編輯器中建立的使用者程式。可多次更新 |

詳細請參考 [程式燒錄_共通步驟](./01-program-setup-common.md)。

---

## 三種燒錄方式

ESP32 可使用三種燒錄方式。

| 方式 | 用途 | 支援開發板 | 特性 |
|------|------|-----------|------|
| **WiFi OTA** | 一般更新 | ESP32 全系列 | 最快、不需傳輸線 |
| **BLE** | AP 限制環境、封閉式裝置 | ESP32（S2 除外） | 不需傳輸線、無 WiFi 也可 |
| **USB** | 首次設定、穩定燒錄 | 所有開發板 | 任何環境都能穩定運作 |

---

## 方式1：WiFi OTA 燒錄（推薦）

最快速的無線燒錄方式。

### 前提條件

- 韌體已透過 USB 燒錄完成
- ESP32 已連接到 WiFi
- DigiCode Finder（桌面應用程式）

### 步驟

1. **啟動 DigiCode Finder**
   - 會自動偵測網路上的 DigiCode 裝置
   - 下載：https://github.com/fablab-westharima/DigiCode-Finder/releases

2. **選擇裝置並複製 IP**
   - 點選目標裝置的「選擇」按鈕
   - IP 位址會複製到剪貼簿

3. **在 DigiCode（瀏覽器）中燒錄**
   - 建立程式
   - 點選「燒錄」按鈕
   - 選擇「WiFi OTA」
   - 在裝置選擇對話框中貼上 IP
   - 「開始燒錄」

詳細請參考 [OTA 設定指南](./05-ota-guide.md)。

---

## 方式2：BLE 燒錄

使用 Bluetooth Low Energy 的燒錄方式。適合無 WiFi 環境或更新封閉式裝置。

### 前提條件

- 韌體已透過 USB 燒錄完成
- 支援 Web Bluetooth 的瀏覽器（Chrome、Edge）

### 支援的開發板

- ESP32（標準）
- ESP32-C3
- ESP32-S3

> **注意：** ESP32-S2 不支援 BLE。

### 步驟

1. **在 DigiCode（瀏覽器）中建立程式**

2. **選擇「燒錄」→「BLE」**

3. **開始藍牙掃描**
   - 點選「搜尋裝置」按鈕
   - 顯示瀏覽器的藍牙選擇對話框

4. **選擇裝置**
   - 選擇名為「DigiCode-XXXXXX」的裝置
   - 點選「配對」

5. **開始燒錄**
   - 顯示進度條
   - 等待完成（比 WiFi OTA 慢）

### 故障排除

| 症狀 | 對策 |
|------|------|
| 找不到裝置 | 重新啟動 ESP32、重新啟動瀏覽器 |
| 配對失敗 | 靠近一點、中斷其他 BLE 連接 |
| 燒錄中途停止 | 重新啟動 ESP32 後重試 |

---

## 方式3：USB 直接燒錄

最穩定的燒錄方式。用於首次設定或故障排除。

### 前提條件

- USB 線（支援資料傳輸）
- USB 驅動程式（CP2102 或 CH340）

### 步驟

1. **使用 USB 線將 ESP32 連接到 PC**

2. **在 DigiCode（瀏覽器）中建立程式**

3. **選擇「燒錄」→「USB」**

4. **選擇序列埠**
   - 顯示瀏覽器的序列埠選擇對話框
   - 選擇 ESP32 連接的連接埠

5. **開始燒錄**
   - 依序燒錄4個檔案（fullPackage 模式）
   - bootloader.bin → partitions.bin → boot_app0.bin → firmware.bin

### 安裝 USB 驅動程式

如果 ESP32 無法識別，需要安裝 USB 驅動程式。

**CP2102 晶片開發板：**
- https://www.silabs.com/developers/usb-to-uart-bridge-vcp-drivers

**CH340 晶片開發板：**
- http://www.wch.cn/downloads/CH341SER_ZIP.html

### 故障排除

| 症狀 | 對策 |
|------|------|
| 連接埠未顯示 | 安裝 USB 驅動程式 |
| 燒錄失敗 | 按住 BOOT 按鈕同時開始燒錄 |
| 逾時錯誤 | 嘗試其他 USB 連接埠 |

## 韌體初次燒錄

首次在新的 ESP32 使用 DigiCode 時，需要燒錄韌體。

### 步驟

1. 點選左側選單的「**韌體燒錄**」
2. 使用 USB 線將 ESP32 連接到 PC
3. 點選「INSTALL」按鈕
4. 選擇序列埠
5. 等待燒錄完成（約1分鐘）

### WiFi 設定（OTA 用）

韌體燒錄後，要使用 WiFi OTA 需要：

1. 點選「WiFi 設定」
2. 選擇序列埠並連接
3. 輸入 SSID 和密碼
4. 點選「連接測試」
5. 成功後會顯示固定 IP 位址

## 使用序列監視器確認運作

序列監視器對程式除錯很有幫助。

### 在瀏覽器中確認

DigiCode 內建序列監視器功能：

1. 使用 USB 連接 ESP32
2. 點選側邊欄的「序列監視器」
3. 選擇連接埠並「連接」
4. 波特率：115200

### 在 Arduino IDE 中確認

1. 工具 → 開啟序列監視器
2. 波特率：115200
3. 顯示 ESP32 的輸出

## 選擇燒錄方式

| 情況 | 推薦方式 |
|------|---------|
| 一般更新（最快） | WiFi OTA |
| 無 WiFi 環境 | BLE |
| 首次設定 | USB |
| 故障排除 | USB |
| 封閉式裝置 | BLE 或 WiFi OTA |
| 教室・工作坊（多台裝置） | WiFi OTA（批次更新） |
