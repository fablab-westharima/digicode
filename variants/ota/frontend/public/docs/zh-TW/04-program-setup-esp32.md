## 術語

| 術語 | 說明 |
|------|------|
| **程式燒錄** | 燒錄用積木編輯器建立的程式的操作（每次） |
| **韌體燒錄** | 燒錄 WiFi OTA / BLE 用基礎軟體的操作（**僅在使用 WiFi OTA / BLE 的第一次**） |

詳細請參閱 [共通步驟](./01-program-setup-common.md)。

## 方式 1: USB 直接燒錄（基本・推薦）

最可靠的燒錄方式。適合初學者到故障排除。

### 前提條件

- USB 線（支援資料傳輸）
- USB 驅動程式（CP2102 或 CH340）

### 安裝 USB 驅動程式

如果 ESP32 未被識別，需要安裝驅動程式。

**CP2102 開發板（許多 ESP32-DevKitC、M5StampS3A 等）:**
- https://www.silabs.com/developers/usb-to-uart-bridge-vcp-drivers

**CH340 開發板（許多通用 ESP32 開發板等）:**
- http://www.wch.cn/downloads/CH341SER_ZIP.html

### 步驟

1. 透過 USB 線連接 ESP32 到電腦
2. 在積木編輯器中建立程式
3. 點擊「**燒錄**」→ 選擇「**USB**」
4. 在瀏覽器的序列埠選擇對話框中選擇 ESP32 的埠
5. 開始燒錄（約 30 秒）→ 完成後自動重新啟動

### 故障排除

| 症狀 | 解決方法 |
|------|---------|
| 埠未顯示 | 安裝 USB 驅動程式 |
| 燒錄失敗 | 按住 BOOT 按鈕的同時開始燒錄 |
| 逾時錯誤 | 嘗試其他 USB 埠或連接線 |

## 方式 2: WiFi OTA（選用）

不需連接線的最快燒錄方式。需要一次性設定。

### 前提條件

- **韌體燒錄完成**（僅第一次，透過 USB 執行）
- WiFi 設定完成（SSID/密碼）
- DigiCode Finder（桌面應用程式）

### 步驟

1. 啟動 DigiCode Finder，自動偵測裝置
2. 點擊目標裝置的「選擇」以複製 IP 位址
3. 點擊「燒錄」→「WiFi OTA」→ 貼上 IP 位址
4. 開始燒錄（約 15 秒）

→ 設定詳細: [OTA 設定指南](./05-ota-guide.md)

---

## 方式 3: BLE（選用）

藍牙燒錄方式。適合更新放入外殼的裝置或沒有 WiFi 的地方。

### 前提條件

- **韌體燒錄完成**（僅第一次，透過 USB 執行）
- 支援 Web Bluetooth 的瀏覽器（Chrome、Edge）

### 支援開發板

所有支援 BLE（supportsBle: true）的 ESP32 系列開發板。

### 步驟

1. 建立程式
2. 點擊「燒錄」→ 選擇「BLE」
3. 點擊「搜尋裝置」，開始藍牙掃描
4. 選擇「DigiCode-XXXXXX」並配對
5. 開始燒錄（約 40 秒）

→ 設定詳細: [OTA 設定指南](./05-ota-guide.md)

## 韌體初始燒錄（僅在使用 WiFi OTA / BLE 時）

想使用 WiFi OTA 或 BLE 時，只需執行一次。

### 步驟

1. 點擊左側選單的「**韌體燒錄**」
2. 透過 USB 線連接 ESP32
3. 點擊「INSTALL」
4. 選擇序列埠
5. 等待完成（約 1 分鐘）

### WiFi 設定（WiFi OTA 用）

韌體燒錄後，要使用 WiFi OTA 需要：

1. 點擊「WiFi 設定」
2. 選擇序列埠並連接
3. 輸入 SSID 和密碼
4. 點擊「連線測試」
5. 成功後顯示固定 IP 位址

---

## 序列監視器確認動作

對程式除錯很方便。

1. 透過 USB 連接 ESP32
2. 點擊側邊欄的「序列監視器」
3. 選擇埠並點擊「連接」
4. 鮑率: 115200

## 燒錄方式的選擇方法

| 情況 | 推薦方式 |
|------|---------|
| 初學者・日常開發 | **USB**（最可靠） |
| 已設定 WiFi OTA | WiFi OTA（最快速） |
| 沒有 WiFi 環境 | USB 或 BLE |
| 放入外殼的裝置 | BLE 或 WiFi OTA |
| 教室批次更新 | WiFi OTA |
| 想節省編譯配額 | 本地編譯伺服器 |
| 故障排除・復原 | **USB**（最可靠） |
