## 基礎積木

用於基本程式結構和 ESP32 I/O 控制的積木。

### 程式結構

| 積木名稱 | 說明 | 備註 |
|----------|------|------|
| **設定** | 程式啟動時執行一次 | 放置初始化程式碼 |
| **迴圈** | 重複執行的主要處理 | 放置感測器讀取和控制邏輯 |

### 數位 I/O

| 積木名稱 | 說明 | 參數 |
|----------|------|------|
| **設定腳位模式** | 設定腳位輸入/輸出模式 | 腳位編號、模式（INPUT/OUTPUT/INPUT_PULLUP） |
| **數位寫入** | 輸出 HIGH/LOW 到腳位 | 腳位編號、值（HIGH/LOW） |
| **數位讀取** | 讀取腳位狀態 | 腳位編號 → true/false |
| **內建 LED 開** | 開啟 ESP32 內建 LED（GPIO2） | — |
| **內建 LED 關** | 關閉 ESP32 內建 LED（GPIO2） | — |

### 類比 I/O

| 積木名稱 | 說明 | 參數 |
|----------|------|------|
| **類比讀取** | 讀取類比值 | 腳位編號（建議 GPIO32-39）→ 0-4095 |
| **PWM 輸出** | 輸出 PWM 訊號 | 腳位編號、工作週期（0-255） |

### 時間與計時

| 積木名稱 | 說明 | 參數 |
|----------|------|------|
| **等待** | 等待指定時間 | 時間（毫秒） |
| **經過時間** | 取得開機後的經過時間（ms） | → 毫秒 |

### 序列通訊

| 積木名稱 | 說明 | 參數 |
|----------|------|------|
| **序列開始** | 初始化序列通訊 | 鮑率（9600/57600/115200） |
| **序列輸出** | 輸出到序列監視器 | 輸出值 |
| **序列輸出行** | 輸出並換行 | 輸出值 |

### 數值轉換

| 積木名稱 | 說明 | 參數 |
|----------|------|------|
| **數值對應** | 將數值映射到不同範圍 | 值、輸入Min/Max、輸出Min/Max → 映射值 |

### ESP32 工具

| 積木名稱 | 說明 | 參數 |
|----------|------|------|
| **Restart ESP32** | 重新啟動 ESP32 | — |
| **Deep Sleep** | 進入深度睡眠（省電） | 秒數（喚醒後從頭執行） |
| **Light Sleep** | 進入輕度睡眠 | 秒數（喚醒後繼續執行） |
| **Free Memory (bytes)** | 取得可用堆積記憶體 | → 位元組 |
| **ESP Chip ID** | 取得晶片唯一 ID | → 字串 |
| **CPU Frequency (MHz)** | 取得 CPU 時脈頻率 | → MHz |
| **播放音調** | 以指定頻率輸出聲音 | 腳位編號、頻率（Hz）、持續時間（ms） |

## 中斷與時刻

腳位中斷、定期計時器及時刻取得的積木。

> **注意：** 中斷處理程式內請勿呼叫 `delay()` 或 `Serial.print()`，否則 ESP32 會當機。請使用 `volatile` 變數設定旗標，並在主迴圈中處理。

### 腳位中斷

| 積木名稱 | 說明 | 參數 |
|----------|------|------|
| **設定中斷** | 腳位變化時呼叫處理程式 | 腳位、模式（RISING/FALLING/CHANGE）、處理程式 |
| **移除中斷** | 移除中斷處理程式 | 腳位 |
| **檢查中斷** | 確認中斷旗標並執行處理 | 放置於迴圈中 |

### 定期計時器（Ticker）

**支援：** 僅限 ESP32（Ticker.h）

| 積木名稱 | 說明 | 參數 |
|----------|------|------|
| **啟動定時器** | 按指定間隔定期呼叫處理程式 | 間隔（ms）、處理程式 |
| **停止定時器** | 停止定期計時器 | — |
| **檢查定時器** | 確認計時器旗標並執行處理 | 放置於迴圈中 |

### NTP 時間同步

**支援：** WiFi 連線的開發板（需要 WiFi 連線）

| 積木名稱 | 說明 | 參數 |
|----------|------|------|
| **NTP同步** | 與 NTP 伺服器同步時間 | 伺服器 URL、時區偏移（秒，JST=32400） |
| **取得Unix時間** | 取得目前 Unix 時間戳記 | → 秒數 |
| **取得格式化時間** | 以指定格式取得時間字串 | 格式（%Y-%m-%d %H:%M:%S 等）→ 字串 |
| **取得時間元件** | 單獨取得年/月/日/時/分/秒 | 項目選擇 → 數字 |

### RTC（即時時鐘）

**程式庫：** RTClib（Dockerfile 內建）

| 積木名稱 | 說明 | 參數 |
|----------|------|------|
| **RTC初始化** | 透過 I2C 初始化 DS3231/DS1307 | 型號選擇 |
| **設定RTC時間** | 設定 RTC 日期時間 | 年/月/日/時/分/秒 |
| **取得RTC** | 取得年/月/日/時/分/秒/Unix時間 | 項目選擇 → 數字 |
| **RTC格式化時間** | 以 "YYYY-MM-DD HH:MM:SS" 格式取得 | → 字串 |

## 相機

拍攝靜態影像、透過 HTTP 傳送，以及 MJPEG 影像串流。

**支援的機型：**
- **ESP32-CAM**（AI-Thinker）
- **XIAO ESP32S3 Sense**（Seeed Studio）

**需要：** 串流和 HTTP 傳送時需要 WiFi 連線

| 積木名稱 | 說明 | 參數 |
|----------|------|------|
| **相機 初始化** | 初始化相機（自動腳位設定） | 開發板型號 → true/false |
| **相機 拍照** | 拍攝照片到內部緩衝區 | → true/false（之後呼叫儲存/傳送積木） |

| **相機 HTTP傳送** | 透過 HTTP POST 傳送拍攝的影像 | → true/false（需要 WiFi） |
| **相機 串流開始** | 啟動 MJPEG 串流伺服器 | 埠號（透過 http://\<IP\>:\<Port\>/stream 存取） |

## 支援的開發板

DigiCode 是以 ESP32 為核心的積木程式編輯器。積木會根據所選開發板的功能自動顯示或隱藏。

| 旗標 | 說明 | 範例開發板 |
|------|------|-----------|
| `supportsWifi` | 可使用 WiFi 積木、HTTP、MQTT 和 HA | ESP32 系列全部、Pico W、Nano RP2040 Connect |
| `supportsOta` | 可使用 WiFi OTA 韌體燒錄（DigiCode OTA） | ESP32 系列（RP2040 不支援） |
| `supportsBle` | 可使用 BLE 積木（NimBLE） | 僅限 ESP32 系列（Pico W 不支援） |

> **注意：** Pico W 支援 WiFi 但不支援 BLE。RFID、CAN 和相機積木僅限 ESP32。
