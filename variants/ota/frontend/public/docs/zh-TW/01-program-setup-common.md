# 程式燒錄 — 共通步驟

**最後更新:** 2026-04-21

DigiCode 是**僅限 ESP32 系列**的積木程式設計環境。本文件說明術語定義和燒錄方式概述。

---

## 術語定義

DigiCode 中「程式燒錄」和「韌體燒錄」是不同的操作。

### 程式燒錄（一般操作）

將用積木編輯器建立的程式燒錄到微控制器的操作。

| 項目 | 內容 |
|------|------|
| **燒錄內容** | 使用者用積木建立的程式 |
| **燒錄方式** | **USB**（基本）/ WiFi OTA / BLE |
| **燒錄頻率** | 任何時候（每次修改程式時） |
| **UI 位置** | 編輯器畫面的「燒錄」按鈕 |

### 韌體燒錄（僅在使用 WiFi OTA / BLE 時）

燒錄 WiFi OTA 或 BLE 所需的基礎軟體的操作。**如果只使用 USB 燒錄，則不需要。**

| 項目 | 內容 |
|------|------|
| **燒錄內容** | OTA 用基礎軟體（直接控制並驅動硬體所需的最小化基礎指令） |
| **燒錄方式** | 僅透過 USB |
| **燒錄頻率** | 使用 WiFi OTA / BLE 的第一次 |
| **UI 位置** | 左側選單「韌體燒錄」 |

---

## 支援的微控制器

DigiCode 僅支援 ESP32 系列。

| 類別 | 開發板範例 | USB | WiFi OTA | BLE |
|------|----------|:---:|:--------:|:---:|
| **Generic ESP32 系列** | ESP32 / S3 / C3 / C6 | ○ | ○ | ○ |
| **M5Stack 系列** | M5Stack Basic / M5StickC Plus / ATOM / M5Stamp 等 | ○ | ○ | ○ |
| **XIAO ESP32 系列** | XIAO ESP32C3 / S3 / C6 | ○ | ○ | ○ |

> **不支援:** ESP8266、Arduino Uno/Nano、RP2040 系列不在 DigiCode 的支援範圍內。

## 4 種燒錄方式

### 1. USB 直接燒錄（基本・推薦）

| 項目 | 內容 |
|------|------|
| **特色** | 最可靠・在任何環境下都能運作 |
| **速度** | 高速（約 30 秒） |
| **連接線** | 需要 |
| **事前準備** | 安裝 USB 驅動程式 |
| **支援開發板** | 所有開發板 |

### 2. WiFi OTA（選用）

| 項目 | 內容 |
|------|------|
| **特色** | 不需連接線・最快速・可同時更新多台裝置 |
| **速度** | 最快（約 15 秒） |
| **連接線** | 不需要 |
| **事前準備** | 韌體燒錄＋WiFi 設定（僅第一次） |
| **支援開發板** | ESP32 系列（supportsOta: true） |
| **所需物品** | WiFi 路由器、DigiCode Finder |

### 3. BLE（選用）

| 項目 | 內容 |
|------|------|
| **特色** | 不需連接線・不需 WiFi |
| **速度** | 中速（約 40 秒） |
| **連接線** | 不需要 |
| **事前準備** | 韌體燒錄（僅第一次） |
| **支援開發板** | 支援 BLE 的 ESP32（supportsBle: true） |
| **所需物品** | 支援 Web Bluetooth 的瀏覽器（Chrome、Edge） |

### 4. 本地編譯伺服器（加速選項）

| 項目 | 內容 |
|------|------|
| **特色** | 不消耗雲端編譯配額・可離線使用 |
| **所需物品** | Docker（或 OrbStack、Rancher Desktop 等） |
| **推薦方案** | Pro 以上 |

→ 詳細: [本地編譯伺服器](./local-compile-server.md)

## 準備物品

### 必要
- 電腦（Windows / Mac / Linux）
- 網頁瀏覽器（推薦 Chrome、Edge）
- 支援的 ESP32 開發板
- USB 線（支援資料傳輸）
- USB 驅動程式（CP2102 或 CH340）

### 選用
- **WiFi OTA 用:** WiFi 路由器、DigiCode Finder
- **BLE 用:** 支援 Web Bluetooth 的瀏覽器（Chrome、Edge）
- **本地編譯用:** Docker 環境

---

## 基本燒錄流程（USB）

1. 透過 USB 線連接 ESP32 到電腦
2. 在積木編輯器中建立程式
3. 點擊「燒錄」→ 選擇「USB」
4. 選擇序列埠
5. 燒錄完成後，ESP32 自動重新啟動並執行程式

---

## 如果想使用 WiFi OTA / BLE

只需一次性設定：

1. 透過 USB 連接 ESP32
2. 左側選單「**韌體燒錄**」→ 點擊「INSTALL」
3. （WiFi OTA 的情況）在「WiFi 設定」中設定 SSID 和密碼
4. 之後可以不需要 USB 連線，使用 WiFi OTA / BLE 燒錄

→ 詳細: [OTA 設定指南](./05-ota-guide.md)

## 常見失敗和解決方法

| 症狀 | 原因 | 解決方法 |
|------|------|---------|
| 埠未顯示 | USB 驅動程式未安裝 | 安裝 CP2102 或 CH340 驅動程式 |
| 燒錄錯誤 | USB 線是充電專用 | 更換支援資料傳輸的連接線 |
| 燒錄中逾時 | 連接不穩定 | 嘗試其他 USB 埠或連接線 |
| 無法進入 BOOT 模式 | 開發板特定步驟 | 按住 BOOT 按鈕的同時開始燒錄 |
| WiFi OTA 未偵測到 | 未連接 WiFi 或未燒錄韌體 | 透過 USB 確認韌體燒錄和 WiFi 設定 |
