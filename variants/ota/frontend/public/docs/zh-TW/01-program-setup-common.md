# 程式燒錄_共通步驟

**最後更新:** 2025-12-28

---

## 簡介

DigiCode 可以將使用 Blockly 建立的程式燒錄到多種類型的微控制器開發板。本文件說明所有微控制器的共通準備工作和基本流程。

---

## 術語定義

明確定義本文件中使用的術語。

### 韌體燒錄

燒錄 DigiCode 基礎系統軟體的作業。此作業**僅需進行一次**。

| 項目 | 內容 |
|------|------|
| **燒錄什麼** | DigiCode 基礎程式（Arduino C++） |
| **燒錄方法** | 僅透過 USB |
| **燒錄頻率** | 僅首次（清除 Flash 後重新設定時除外） |
| **UI 位置** | 左側選單「韌體燒錄」 |

### 程式燒錄

將使用者使用 Blockly 建立的程式燒錄到微控制器的作業。在韌體燒錄完成的前提下，可以重複燒錄、更新多次。

| 項目 | 內容 |
|------|------|
| **燒錄什麼** | 在積木編輯器中建立的使用者程式 |
| **燒錄方法** | WiFi OTA / BLE / USB |
| **燒錄頻率** | 任意次數（每次程式變更時） |
| **UI 位置** | 編輯器畫面「燒錄」按鈕 |

## 三種燒錄方式

### WiFi OTA（Over-The-Air）

透過 WiFi 無線燒錄程式的方式。

| 項目 | 內容 |
|------|------|
| **優點** | 快速、不需傳輸線 |
| **缺點** | 首次需燒錄韌體 |
| **支援開發板** | ESP32 系列 |
| **需求** | DigiCode Finder（裝置偵測應用程式） |

### BLE（Bluetooth Low Energy）

透過藍牙燒錄程式的方式。

| 項目 | 內容 |
|------|------|
| **優點** | 不需傳輸線、沒有 WiFi 也可使用 |
| **缺點** | 比 WiFi OTA 慢 |
| **支援開發板** | ESP32（S2 除外） |
| **需求** | 支援 Web Bluetooth 的瀏覽器（Chrome、Edge） |

### USB 直接燒錄

使用 USB 線連接燒錄的方式。

| 項目 | 內容 |
|------|------|
| **優點** | 穩定、任何環境都能使用 |
| **缺點** | 需要連接傳輸線 |
| **支援開發板** | 所有開發板 |
| **需求** | USB 線、驅動程式 |

## 準備事項

### 必要
- 電腦（Windows / Mac / Linux）
- 瀏覽器（建議 Chrome、Edge）
- 對應的微控制器開發板
- USB 線（首次韌體燒錄用）

### 選用
- **WiFi OTA 用:** WiFi 路由器、DigiCode Finder
- **BLE 用:** 支援 Web Bluetooth 的瀏覽器

## 韌體燒錄的基本流程

所有微控制器在首次使用時都需要燒錄**韌體**。

### 步驟

1. **連接 PC 與 USB**
   - 使用 USB 線將微控制器連接到 PC
   - 確認開發板已被識別

2. **在 DigiCode 燒錄韌體**
   - 點選左側選單「韌體燒錄」
   - 點選「INSTALL」按鈕
   - 選擇序列埠
   - 等待燒錄完成（約1分鐘）

3. **確認**
   - 裝置重新啟動並正常運作

### 在 ESP32 使用 WiFi OTA 時

韌體燒錄後需要額外設定：

1. 點選「WiFi 設定」
2. 選擇序列埠並連接
3. 輸入 SSID 和密碼
4. 點選「連接測試」
5. 連接成功後會顯示固定 IP 位址

## 確認裝置偵測

### ESP32 系列（使用 WiFi OTA 時）

**使用 DigiCode Finder（推薦）：**

1. 下載 DigiCode Finder
   - https://github.com/fablab-westharima/DigiCode-Finder/releases
2. 啟動應用程式
3. 會自動偵測同一網路上的 DigiCode 裝置

### 所有微控制器共通（USB 連接時）

1. 將 ESP32 連接 USB
2. 點選側邊欄的「序列監視器」
3. 選擇連接埠並連接
4. 顯示輸出表示正常

---

## 常見失敗與對策

| 症狀 | 原因 | 對策 |
|------|------|------|
| 裝置無法識別 | USB 連接不良 | 重新連接傳輸線、嘗試其他連接埠 |
| 燒錄時發生錯誤 | 驅動程式不足 | 安裝 USB 驅動程式 |
| 韌體燒錄後仍無法運作 | 裝置選擇錯誤 | 確認開發板資訊並重置 |
| WiFi OTA 無法偵測 | WiFi 未連接 | 透過 USB 連接確認 WiFi 設定 |
