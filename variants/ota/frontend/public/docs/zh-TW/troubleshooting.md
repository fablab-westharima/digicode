# 故障排除

**最後更新：** 2026-04-21

整理 DigiCode 的常見問題與解決方法。

---

## 程式燒錄相關

### 無法辨識 ESP32

**症狀：** 序列埠選擇對話框中沒有顯示 ESP32

**解決方法：**

1. 確認 USB 線支援**資料傳輸**（充電專用線不可用）
2. 安裝 ESP32 USB 驅動程式
   - **CP2102**：https://www.silabs.com/developers/usb-to-uart-bridge-vcp-drivers
   - **CH340**：http://www.wch.cn/downloads/CH341SER_ZIP.html
3. 嘗試其他 USB 連接埠
4. 重新插拔 ESP32 的 USB 線
5. 切換至 Chrome / Edge（需要 Web Serial API）

### 燒錄時發生錯誤

**症狀：** 顯示「Failed to connect」等錯誤訊息

**解決方法：**

1. 按住 ESP32 的 **BOOT 按鈕**後開始燒錄
2. 燒錄過程中持續按住 BOOT 按鈕
3. 確認沒有其他應用程式正在使用該連接埠（如序列監視器）
4. 重新啟動瀏覽器後重試

### 超過編譯配額

**症狀：** 顯示「本月編譯次數已達上限」

**解決方法：**

1. 等待下個月
2. 升級至更高方案（Lite / Pro / Enterprise）
3. 使用[本地編譯伺服器](./local-compile-server.md)不會消耗雲端配額

## WiFi OTA 相關

### WiFi OTA 找不到裝置

**症狀：** DigiCode Finder 無法偵測到裝置

**解決方法：**

1. 確認 ESP32 與 PC **連接相同的 WiFi 網路**
2. 確認已安裝 **Bonjour Print Services**（僅限 Windows）
3. 確認防火牆沒有封鎖 mDNS（連接埠 5353）
4. 在 ESP32 的序列監視器中確認 WiFi 連線成功與 IP 位址
5. 重新啟動 ESP32（RESET 按鈕）

### OTA 更新中途停止

**症狀：** 更新在 20% 左右停止

**解決方法：**

1. WiFi 訊號可能過弱 → 將 ESP32 移至靠近路由器處
2. 重新啟動 ESP32 後重試
3. 透過 USB 重新燒錄韌體後，再試一次 WiFi OTA

### WiFi OTA 燒錄後裝置無回應

**症狀：** 燒錄成功，但程式無法執行

**解決方法：**

1. 透過 USB 重新燒錄韌體
2. 「清除完整快閃記憶體（除錯用）」→ 韌體燒錄 → WiFi 設定

## BLE 相關

### BLE 找不到裝置

**症狀：** 按下「搜尋裝置」按鈕後沒有顯示裝置

**解決方法：**

1. **瀏覽器支援** — 確認使用 Chrome / Edge（需要 Web Bluetooth）
2. **開發板支援** — ESP32-S2 不支援 BLE；RP2040 Pico W 也不支援 BLE
3. 重新啟動 ESP32
4. 重新啟動瀏覽器

### BLE 配對失敗

**症狀：** 找到裝置但配對失敗

**解決方法：**

1. 將 ESP32 放置在電腦附近
2. 暫時斷開其他 BLE 連線（滑鼠、鍵盤等）
3. 重新啟動 ESP32 後重試

### macOS 上顯示舊裝置名稱

**症狀：** 更改裝置名稱後，macOS 的瀏覽器仍顯示舊名稱

**解決方法：** 這是 macOS BLE 快取造成的 OS 層級問題，DigiCode 端無法修正。Windows 下正常運作，Mac 使用者可能受影響。

## 程式執行相關

### 程式無法執行

**症狀：** 燒錄成功，但程式無法運作

**解決方法：**

1. 按下 ESP32 的 RESET 按鈕重新啟動
2. 在序列監視器中確認錯誤訊息
3. 確認腳位編號是否正確（可用腳位因開發板而異）
4. 重新確認感測器 / 馬達的接線

### LED 不閃爍

**症狀：** 燒錄了 LED 閃爍程式，但 LED 沒有閃爍

**解決方法：**

1. **GPIO2（內建 LED）的情況**：
   - 部分開發板沒有內建 LED
   - 嘗試連接外部 LED
2. **外部 LED 的情況**：
   - 確認極性（正負極）
   - 串聯電阻（330Ω〜1kΩ）
   - 確認腳位編號

## 班級功能相關

### 無法透過邀請 URL 加入班級

**症狀：** 開啟邀請 URL 後無法加入班級

**解決方法：**

1. 確認已登入 DigiCode（未登入時請登入後再次開啟 URL）
2. 確認邀請 URL 是否已過期（請詢問教師）
3. 確認該邀請 URL 是否已被其他帳號使用

### 作業沒有顯示

**症狀：** 已加入班級但作業沒有顯示

**解決方法：**

1. 重新整理頁面
2. 確認班級伺服器（`class.digital-fab.jp`）的運作狀態
3. 詢問教師作業是否已發佈

### 無法提交答案

**症狀：** 按下答案提交按鈕時出現錯誤

**解決方法：**

1. 確認網路連線
2. 班級伺服器可能暫時停機 → 稍後再試
3. 確認 Blockly XML 是否過大（一般不會有問題）

## 方案與付款相關

### 無法變更方案

**症狀：** Stripe 客戶入口網站無法開啟或出現錯誤

**解決方法：**

1. 確認彈出視窗封鎖程式未啟用
2. 確認瀏覽器的 Cookie 已啟用
3. 確認 Stripe 的伺服器狀態

### 受邀帳號自行訂閱方案時

**症狀：** 已從 Fablab Nishiharima 獲得 Enterprise 等級權限，但自行訂閱了 Pro 方案

**解決方法：** 自行訂閱後，受邀授予的權限將**被移除**。此後將套用您自己訂閱的方案。訂閱前請仔細確認確認對話框的內容。

## 瀏覽器相關

### Blockly 編輯器沒有顯示

**症狀：** 編輯器頁面全白，或積木沒有顯示

**解決方法：**

1. 清除瀏覽器快取
2. 將瀏覽器更新至最新版本
3. 嘗試其他瀏覽器（Chrome、Edge）
4. 確認 JavaScript 已啟用

### 顯示編譯錯誤

**症狀：** 出現「編譯錯誤」對話框

**解決方法：**

1. 確認錯誤訊息內容
2. 確認積木連接是否正確
3. 確認所有必要參數是否已填寫
4. 嘗試範例專案以縮小問題範圍

---

## 其他

### 無法儲存專案

**症狀：** 按下儲存按鈕時發生錯誤

**解決方法：**

1. 確認已登入（訪客只能儲存至本機）
2. 確認網路連線
3. 確認瀏覽器的本機儲存空間已啟用
4. 重新啟動瀏覽器

### 無法載入範例專案

**症狀：** 選擇範例後，編輯器沒有反映

**解決方法：**

1. 重新整理頁面
2. 登出後重新登入
3. 嘗試其他範例
