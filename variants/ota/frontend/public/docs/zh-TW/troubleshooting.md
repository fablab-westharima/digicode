# 故障排除

整理 DigiCode 的常見問題與解決方法。

## 韌體燒錄相關

### ESP32 無法識別

**症狀:** 序列埠沒有顯示任何東西

**解決方法:**
1. 確認 USB 線是否為**資料傳輸專用**（充電專用線無法使用）
2. 安裝 ESP32 的 USB 驅動程式
   - CP2102 驅動程式: https://www.silabs.com/developers/usb-to-uart-bridge-vcp-drivers
   - CH340 驅動程式: http://www.wch.cn/downloads/CH341SER_ZIP.html
3. 嘗試其他 USB 連接埠
4. 重新啟動 ESP32 電源（拔插 USB 線）

### 燒錄時發生錯誤

**症狀:** 顯示「Failed to connect」等錯誤

**解決方法:**
1. 按住 ESP32 的 **BOOT 按鈕**同時開始燒錄
2. 燒錄期間持續按住 BOOT 按鈕
3. 確認序列監視器等其他應用程式是否正在使用該連接埠
4. 重新啟動瀏覽器後重試

## 程式運作相關

### 程式不執行

**症狀:** 燒錄成功但程式不執行

**解決方法:**
1. 按下 ESP32 的 RESET 按鈕重新啟動
2. 在序列監視器確認錯誤訊息
3. 確認腳位編號是否正確（不同開發板可用的腳位不同）
4. 重新確認感測器、馬達的接線

### LED 不閃爍

**症狀:** 燒錄了 LED 閃爍程式但 LED 不閃爍

**解決方法:**
1. GPIO2（內建 LED）的情況：
   - 部分開發板沒有內建 LED
   - 嘗試連接外部 LED
2. 外部 LED 的情況：
   - 確認極性（正負）
   - 串聯電阻（330Ω〜1kΩ）
   - 確認腳位編號

## WiFi OTA 相關

### 無法進行 WiFi OTA 更新

**症狀:** 掃描找不到裝置或無法連接

**解決方法:**
1. 確認 ESP32 和電腦是否連接到**相同的 WiFi 網路**
2. 確認防火牆是否封鎖 mDNS（連接埠 5353）
3. 在 ESP32 的序列監視器確認 WiFi 連接成功
4. 重新啟動 ESP32（按 RESET 按鈕）

### OTA 更新中途停止

**症狀:** 更新在約 20% 時停止

**解決方法:**
1. WiFi 訊號可能較弱 → 將 ESP32 移近路由器
2. 重新啟動 ESP32 後重試
3. 透過 USB 燒錄後再次嘗試 WiFi OTA

## 瀏覽器相關

### Blockly 編輯器不顯示

**症狀:** 編輯器頁面全白或積木不顯示

**解決方法:**
1. 清除瀏覽器快取
2. 將瀏覽器更新到最新版本
3. 嘗試其他瀏覽器（Chrome、Edge、Safari）
4. 確認 JavaScript 是否已啟用

### 顯示編譯錯誤

**症狀:** 顯示「編譯錯誤」對話框

**解決方法:**
1. 確認錯誤訊息
2. 確認積木的連接是否正確
3. 確認所有必要參數是否已輸入
4. 從範例專案嘗試，分離問題

## 其他

### 無法儲存專案

**症狀:** 按下儲存按鈕但發生錯誤

**解決方法:**
1. 確認是否已登入
2. 確認網路連線
3. 確認瀏覽器的本地儲存是否已啟用
4. 重新啟動瀏覽器

### 無法載入範例專案

**症狀:** 選擇範例但沒有反映到編輯器

**解決方法:**
1. 重新整理頁面
2. 登出後再重新登入
3. 嘗試其他範例
