## 帳號與登入

### Q：需要註冊帳號嗎？

**A：** 不需要，**以訪客身份也可以建立程式並進行燒錄**。不過若要將專案儲存至雲端，則需要註冊帳號（從 Free 方案即可使用）。

### Q：忘記密碼了

**A：** 在登入畫面點選「忘記密碼？」連結，即可將重設連結傳送至您的已註冊電子郵件。

### Q：可以從多台裝置登入嗎？

**A：** 可以，使用同一帳號可從多台裝置登入。專案儲存於雲端，可從任何裝置存取。

## 編輯器與積木

### Q：找不到積木

**A：** 請確認以下項目：

1. **機器人模式** — 是否已選擇正確的模式？DigiCode 共有 7 種模式：
   - Humanoid（雙足步行）
   - Wheel（輪型）
   - Transform（變形機器人）
   - Home Assistant（IoT 整合）
   - Generic（通用）
   - All Blocks（所有積木）
   - Custom（自訂）

2. **類別** — 請展開工具箱中的各個類別

3. **開發板支援** — 部分積木會依據所選開發板而隱藏
   - 例：BLE 積木在 Pico W 上不顯示（不支援 BLE）

### Q：無法連接積木

**A：** 積木有固定的連接形狀：

- **凹凸形狀** — 上下連接的積木（執行順序）
- **圓孔** — 輸入數值的積木（數字、字串等）
- **三角形** — 輸入條件的積木（布林值）

形狀不符的積木無法互相連接。

### Q：切換開發板後積木消失了

**A：** 切換開發板時，不支援的積木可能會從工作區移除（例如：Pico W 上的 BLE 積木）。切換前請先儲存專案。

### Q：日文字元出現亂碼

**A：** 在序列通訊中使用日文時可能出現亂碼。請確認序列監視器的鮑率是否一致（通常為 115200）。

## 程式燒錄

### Q：「程式燒錄」與「韌體燒錄」有什麼差別？

**A：**

| 操作 | 內容 | 頻率 |
|------|------|------|
| **程式燒錄** | 將積木編輯器中建立的程式燒錄至開發板 | 每次（USB / WiFi OTA / BLE） |
| **韌體燒錄** | 燒錄 WiFi OTA / BLE 所需的基礎軟體 | **僅限首次使用 WiFi OTA / BLE 時** |

**若只使用 USB 燒錄，則不需要進行韌體燒錄。**

### Q：推薦哪種燒錄方式？

**A：** 初學者與一般開發建議使用 **USB 直接燒錄**，最為穩定可靠。熟悉後若想無線更新，再進行 WiFi OTA / BLE 設定。

### Q：編譯失敗

**A：** 常見原因與解決方法：

| 錯誤 | 原因 | 解決方法 |
|------|------|---------|
| 找不到函式庫 | 尚未安裝必要函式庫 | 伺服器端會自動安裝，請重試 |
| 語法錯誤 | 積木連接錯誤 | 確認是否有未連接的積木 |
| 記憶體不足 | 程式太大 | 刪除不必要的積木 |
| 超過編譯配額 | 已達到方案上限 | 升級方案或使用本地編譯伺服器 |

### Q：無法辨識 ESP32

**A：** 請依序確認以下項目：

1. **USB 線** — 是否支援資料傳輸（充電專用線不可用）
2. **驅動程式** — 是否已安裝 CP2102 / CH340 驅動程式
3. **連接埠** — 嘗試其他 USB 連接埠
4. **瀏覽器** — 是否使用 Chrome / Edge（需要 Web Serial API）

### Q：燒錄時出現錯誤

**A：** 常見錯誤與解決方法：

- **「A fatal error occurred: Failed to connect」**
  → 按住 ESP32 的 BOOT 按鈕後開始燒錄
- **「Timed out waiting for packet header」**
  → 降低鮑率後重試
- **「No serial data received」**
  → 確認 USB 線與連接埠

### Q：編譯需要很長時間

**A：** 首次編譯約需 30 秒至 1 分鐘。若需加快速度，請使用[本地編譯伺服器](./local-compile-server.md)（建議 Pro 方案以上）。

## WiFi OTA / BLE

### Q：找不到裝置（WiFi OTA）

**A：** 請確認以下項目：

1. **相同 WiFi 網路** — PC 與 ESP32 是否連接相同網路
2. **防火牆** — mDNS（連接埠 5353）是否被封鎖
3. **Bonjour**（僅限 Windows）— 是否已安裝 Bonjour Print Services
4. **ESP32 WiFi 連線** — 是否可在序列監視器中看到 IP 位址

### Q：BLE 找不到裝置

**A：**

1. **瀏覽器支援** — 是否使用 Chrome / Edge（需要 Web Bluetooth）
2. **開發板支援** — ESP32-S2 不支援 BLE；RP2040 Pico W 也不支援 BLE
3. **macOS** — macOS 的 BLE 快取可能顯示舊的裝置名稱（OS 層級問題）。Windows 下正常運作，Mac 可能受影響

### Q：OTA 更新中途停止

**A：**

1. 在 WiFi 訊號強的地方執行
2. 將 ESP32 放置在靠近路由器的位置
3. 暫時斷開其他 WiFi 裝置
4. 重新啟動路由器

## 硬體

### Q：應該使用哪款 ESP32 開發板？

**A：** DigiCode 專為 ESP32 系列設計。以下開發板已確認可正常使用：

- **ESP32-DevKitC** — 最通用
- **M5StampS3A** — DigiCode 推薦主控板（專用擴充板開發中）
- **ESP32-S3** — 高效能
- **XIAO ESP32S3** — 支援攝影機

詳情請參閱[推薦硬體](./recommended-hardware.md)。

### Q：伺服馬達無法動作

**A：**

1. **電源** — 伺服馬達電流需求大，USB 供電（500mA）可能不足。請使用外部電源（5V/2A 以上）
2. **腳位編號** — 確認腳位設定是否正確
3. **初始化** — 是否已放置伺服馬達連接積木

### Q：感測器數值異常

**A：**

1. **接線** — 確認 VCC / GND / 訊號線是否正確連接
2. **電壓** — 確認感測器的工作電壓（3.3V / 5V）
3. **類比腳位** — 是否使用支援 ADC 的腳位（GPIO32-39）

### Q：Humanoid 機器人無法正常行走

**A：**

1. **校準** — 使用 Home Position 積木確認直立姿勢
2. **伺服馬達脈衝寬度** — 在腳位設定頁面調整脈衝寬度（500-2400μs）
3. **速度** — 降低速度以提升穩定性
4. **重心** — 調整電池位置

## 專案管理

### Q：無法儲存專案

**A：**

1. 是否已登入（訪客只能儲存至本機）
2. 是否已輸入專案名稱
3. 網路連線是否正常

### Q：專案消失了

**A：**

1. 是否以正確帳號登入
2. 在專案清單中搜尋
3. 誤刪的專案無法還原

### Q：可以分享專案嗎？

**A：** 使用班級功能，教師可將作業分發給學生、學生可提交答案（Enterprise 方案）。一般公開分享功能預計未來開發。

### Q：可以編輯範例專案嗎？

**A：** 載入範例專案時，會自動建立副本。原始範例不會被修改。

## 班級功能

### Q：什麼是班級功能？

**A：** 專為學校及教育機構設計的班級管理功能（**Enterprise 方案**）。

| 角色 | 可執行操作 |
|------|-----------|
| **教師** | 建立班級、分發作業、查看答案、複製班級 |
| **學生** | 加入班級（邀請 URL）、提交作業、查看提交紀錄 |

### Q：學生需要註冊帳號嗎？

**A：** 需要，學生也必須擁有 DigiCode 帳號。透過教師傳送的邀請 URL 加入班級後，將獲得 Enterprise 等級的權限（由 Fablab Nishiharima 授予）。

### Q：哪個方案可使用班級功能？

**A：** 班級功能為 **Enterprise 方案** 專屬。個人的 Free / Lite / Pro 方案無法使用。

### Q：作業的 Blockly XML 儲存在哪裡？

**A：** 作業與答案的 Blockly XML 儲存於 HPE ML30 上的班級伺服器（`class.digital-fab.jp`）。D1 只儲存基本管理資訊（班級 ID、成員等）。

### Q：班級伺服器停機時會怎樣？

**A：** 班級功能（作業分發、答案提交）將無法使用，但 DigiCode 的一般程式建立與燒錄仍可繼續。系統設計將影響範圍限制在作業與答案操作。

## 方案與費用

### Q：可以免費使用嗎？

**A：** 可以，**Free 方案** 可免費使用基本功能。

| 方案 | 適合對象 | 月費 | 雲端編譯 | 班級功能 |
|------|---------|:----:|:-------:|:-------:|
| **Free** | 試用 / 本地編譯 | ¥0 | 每月 50 次 | — |
| **Lite** | 個人玩家 | — | 每月 250 次 | — |
| **Pro** | 開發者 / Maker | — | 每月 500 次 | — |
| **Enterprise** | 教育機構 / 團隊 | — | 無限制 | ○ |

最新費用請參閱[方案頁面](/plan)。

### Q：如何變更方案？

**A：** 點選右上角帳號選單 → 「方案與付款」即可變更。可透過 Stripe 客戶入口網站管理帳單資訊、變更方案或取消訂閱。

### Q：什麼是受邀帳號？

**A：** 由 Fablab Nishiharima 授予 Enterprise 等級權限的帳號。不需要自行訂閱方案。但您也可以自行訂閱更高方案（此時受邀權限將被移除）。

### Q：有教育機構授權嗎？

**A：** **Enterprise 方案** 提供教育機構所需的班級功能。如需大量授權或特別配套，請聯絡 [Fablab Nishiharima](https://fablab-westharima.com/)。

### Q：使用本地編譯伺服器會消耗雲端配額嗎？

**A：** 不會，使用本地編譯伺服器不會消耗雲端編譯配額。非常適合在 Pro / Enterprise 方案下進行高速、無限制的編譯。→ [本地編譯伺服器](./local-compile-server.md)

## 其他

### Q：支援哪些瀏覽器？

**A：**

| 瀏覽器 | 支援狀況 | 備註 |
|--------|---------|------|
| Chrome | ○ 推薦 | 支援 Web Serial / Web Bluetooth |
| Edge | ○ | 支援 Web Serial / Web Bluetooth |
| Firefox | △ | 不支援 USB / BLE 燒錄 |
| Safari | △ | 不支援 USB / BLE 燒錄 |

使用 USB / BLE 燒錄時，請使用 Chrome 或 Edge。

### Q：可以在智慧型手機上使用嗎？

**A：** 積木編輯器可在智慧型手機上顯示，但不支援 USB 燒錄。WiFi OTA 燒錄可從智慧型手機執行。

### Q：可以離線使用嗎？

**A：** 目前需要網路連線。使用本地編譯伺服器時，編譯本身可離線運作。

### Q：發現了錯誤

**A：** 請在 [GitHub Issues](https://github.com/fablab-westharima/DigiCode/issues) 回報，並附上以下資訊：

- 瀏覽器與版本
- 作業系統
- 使用的方案
- 重現步驟
- 錯誤訊息（如有）
