## 帳號與登入

### Q: 需要註冊帳號嗎？

**A:** 是的，要使用專案儲存和編譯功能需要註冊帳號。免費方案也可使用基本功能。

### Q: 忘記密碼了

**A:** 目前密碼重設功能正在開發中。請透過聯絡表單與我們聯繫。

### Q: 可以在多個裝置登入嗎？

**A:** 可以，同一帳號可以從多個裝置登入。專案儲存在雲端，可從任何裝置存取。

## 編輯器與積木

### Q: 找不到積木

**A:** 請確認以下事項：

1. **機器人模式** - 是否選擇了符合您使用的機器人模式？
   - Humanoid（雙足步行） → 顯示 Humanoid 積木
   - Wheel → 顯示 Wheel 積木
   - 自訂 → 顯示所有積木

2. **類別** - 請展開並確認工具箱中的各個類別

3. **開發板選擇** - 可用的積木會根據選擇的微控制器開發板而變化

### Q: 積木無法連接

**A:** 積木有固定可連接的形狀：

- **凹凸形狀** - 上下連接的積木（執行順序）
- **圓形孔** - 輸入值的積木（數值、字串等）
- **三角形** - 輸入條件的積木（布林值）

形狀不符的積木無法連接。

### Q: 切換開發板後積木消失了

**A:** 切換微控制器開發板時，不相容的積木可能會從工作區移除。請在切換前儲存專案。

### Q: 日文出現亂碼

**A:** 在序列通訊中使用日文時可能會出現亂碼。請確認序列監視器的波特率是否一致（通常為 115200）。

## 編譯與燒錄

### Q: 編譯失敗

**A:** 常見原因與解決方法：

| 錯誤 | 原因 | 解決方法 |
|------|------|----------|
| 找不到函式庫 | 必要函式庫未安裝 | 伺服器會自動安裝，請重試 |
| 語法錯誤 | 積木連接錯誤 | 確認是否有未連接的積木 |
| 記憶體不足 | 程式太大 | 刪除不需要的積木 |

### Q: ESP32 無法識別

**A:** 請按順序確認以下事項：

1. **USB 線** - 是否使用支援資料傳輸的線？（充電專用線無法使用）
2. **驅動程式** - 是否已安裝 CP2102/CH340 驅動程式？
3. **連接埠** - 請嘗試其他 USB 連接埠
4. **瀏覽器** - 是否使用 Chrome/Edge？（需支援 Web Serial API）

### Q: 燒錄時出現錯誤

**A:** 常見錯誤與解決方法：

- **「A fatal error occurred: Failed to connect」**
  → 按住 ESP32 的 BOOT 按鈕同時開始燒錄

- **「Timed out waiting for packet header」**
  → 降低波特率重試（115200 → 57600）

- **「No serial data received」**
  → 確認 USB 線和連接埠

### Q: 編譯時間很長

**A:** 首次編譯可能需要 30 秒至 1 分鐘。第二次以後會透過快取加速。

## WiFi OTA 更新

### Q: 找不到裝置

**A:** 請確認以下事項：

1. **相同 WiFi 網路** - 電腦和 ESP32 是否連接到相同網路？
2. **防火牆** - mDNS（連接埠 5353）是否被封鎖？
3. **ESP32 WiFi 連線** - 可以在序列監視器確認 IP 位址嗎？

### Q: OTA 更新中途停止

**A:** 請嘗試以下方法：

1. 在 WiFi 訊號強的地方執行
2. 將 ESP32 放在靠近電源的位置
3. 暫時中斷其他 WiFi 裝置
4. 重新啟動路由器

### Q: OTA 更新和 USB 燒錄有什麼不同？

**A:**

| 方式 | 優點 | 缺點 |
|------|------|------|
| **USB 燒錄** | 穩定、快速 | 需要連接傳輸線 |
| **WiFi OTA** | 無線、可遠端更新 | 需要初始設定、依賴 WiFi 環境 |

建議首次用 USB 燒錄 OTA 韌體，之後使用 OTA 更新。

## 硬體

### Q: 應該使用哪款 ESP32 開發板？

**A:** 以下開發板已驗證可使用：

- **ESP32-DevKitC** - 最常見，推薦
- **ESP32-WROOM-32** - 標準模組
- **ESP32-S3** - 部分功能有限制

### Q: 伺服馬達不動

**A:** 請確認以下事項：

1. **電源** - 伺服馬達耗電量大，USB 供電可能不足。請使用外部電源（5V/2A 以上）
2. **腳位編號** - 確認腳位設定是否正確
3. **初始化** - 是否已放置伺服連接積木？

### Q: 感測器數值異常

**A:** 請確認以下事項：

1. **接線** - VCC/GND/訊號線是否正確連接？
2. **電壓** - 確認感測器的工作電壓（3.3V/5V）
3. **類比腳位** - 是否使用 ADC 對應腳位（GPIO32-39）？

### Q: Humanoid 機器人走不好

**A:** 請嘗試以下方法：

1. **校正** - 使用原點位置積木確認直立姿勢
2. **伺服脈衝寬度** - 在腳位設定頁面調整脈衝寬度（500-2400μs）
3. **速度** - 降低速度確認穩定性
4. **重心** - 調整電池位置

## 專案管理

### Q: 無法儲存專案

**A:** 請確認以下事項：

1. 是否已登入？
2. 是否已輸入專案名稱？
3. 網路連線是否正常？

### Q: 專案消失了

**A:** 專案儲存在雲端。請確認以下事項：

1. 是否使用正確的帳號登入？
2. 請在專案列表中搜尋
3. 如果誤刪，無法恢復

### Q: 可以分享專案嗎？

**A:** 目前專案分享功能正在開發中。請等待未來更新。

### Q: 可以編輯範例專案嗎？

**A:** 載入範例專案時會自動建立副本。原始範例不會被更改。

## 費用與方案

### Q: 可以免費使用嗎？

**A:** 是的，基本功能可以免費使用。

**免費方案包含：**
- 基本積木
- 專案儲存（有上限）
- 編譯與燒錄
- 範例專案

### Q: 什麼是進階功能？

**A:** 目前所有功能都可免費使用。未來以下功能可能會成為付費功能：

- 進階感測器積木
- PID 調校功能
- 無限專案儲存
- 優先支援

## 其他

### Q: 支援哪些瀏覽器？

**A:** 以下瀏覽器已驗證可使用：

| 瀏覽器 | 支援狀態 | 備註 |
|--------|----------|------|
| Chrome | ○ 推薦 | 支援 Web Serial API |
| Edge | ○ | 支援 Web Serial API |
| Firefox | △ | 不支援 USB 燒錄 |
| Safari | △ | 不支援 USB 燒錄 |

使用 USB 燒錄功能時，請使用 Chrome 或 Edge。

### Q: 可以在手機上使用嗎？

**A:** 積木編輯器可以在手機上使用，但無法使用 USB 燒錄功能。透過 WiFi OTA 更新的話，可以從手機更新程式。

### Q: 可以離線使用嗎？

**A:** 目前需要網路連線。離線版包含在未來開發計畫中。
