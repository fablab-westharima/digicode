# プログラム書き込み — 共通手順

**最終更新:** 2026-04-21

DigiCodeは**ESP32系専用**のブロックプログラミング環境です。本ドキュメントでは用語の定義と書き込み方式の概要を説明します。

---

## 用語の定義

DigiCodeでは、「プログラム書き込み」と「ファームウェア書き込み」は異なる操作です。

### プログラム書き込み（通常の操作）

ブロックエディタで作成したプログラムをマイコンに書き込む操作。

| 項目 | 内容 |
|------|------|
| **何を書き込むか** | ユーザーがブロックで作ったプログラム |
| **書き込み方法** | **USB**（基本）/ WiFi OTA / BLE |
| **書き込み頻度** | 何度でも（プログラムを変えるたびに） |
| **UI上の場所** | エディタ画面「書き込み」ボタン |

### ファームウェア書き込み（WiFi OTA / BLE使用時のみ）

WiFi OTA または BLE を使うために必要な基盤ソフトウェアを書き込む操作。**USBだけでプログラムを書き込む場合は不要です。**

| 項目 | 内容 |
|------|------|
| **何を書き込むか** | OTA用の基盤ソフトウェア（ハードウェアを直接制御し動かすための最小限かつ基盤となる命令） |
| **書き込み方法** | USB経由のみ |
| **書き込み頻度** | WiFi OTA / BLE 使用時の初回のみ |
| **UI上の場所** | 左メニュー「ファームウェア書き込み」 |

---

## 対応マイコン

DigiCodeはESP32系専用です。

| カテゴリ | ボード例 | USB | WiFi OTA | BLE |
|---------|--------|:---:|:--------:|:---:|
| **Generic ESP32系** | ESP32 / S3 / C3 / C6 | ○ | ○ | ○ |
| **M5Stack系** | M5Stack Basic / M5StickC Plus / ATOM / M5Stamp 等 | ○ | ○ | ○ |
| **XIAO ESP32系** | XIAO ESP32C3 / S3 / C6 | ○ | ○ | ○ |

> **非対応:** ESP8266、Arduino Uno/Nano、RP2040系はDigiCodeの対応対象外です。

## 4つの書き込み方式

### 1. USB直接書き込み（基本・推奨）

| 項目 | 内容 |
|------|------|
| **特徴** | 最も確実・どの環境でも動作 |
| **速度** | 高速（約30秒） |
| **ケーブル** | 必要 |
| **事前準備** | USBドライバーのインストール |
| **対応ボード** | 全ボード |

### 2. WiFi OTA（オプション）

| 項目 | 内容 |
|------|------|
| **特徴** | ケーブル不要・最速・複数台同時更新可 |
| **速度** | 最速（約15秒） |
| **ケーブル** | 不要 |
| **事前準備** | ファームウェア書き込み＋WiFi設定（初回のみ） |
| **対応ボード** | ESP32系全般（supportsOta: true） |
| **必要なもの** | WiFiルータ、DigiCode Finder |

### 3. BLE（オプション）

| 項目 | 内容 |
|------|------|
| **特徴** | ケーブル不要・WiFi環境不要 |
| **速度** | 中速（約40秒） |
| **ケーブル** | 不要 |
| **事前準備** | ファームウェア書き込み（初回のみ） |
| **対応ボード** | BLE対応ESP32系（supportsBle: true） |
| **必要なもの** | Web Bluetooth対応ブラウザ（Chrome、Edge） |

### 4. ローカルコンパイルサーバー（高速化オプション）

| 項目 | 内容 |
|------|------|
| **特徴** | クラウドコンパイル枠を使わない・オフライン利用可 |
| **必要なもの** | Docker（またはOrbStack、Rancher Desktop等） |
| **推奨プラン** | Pro以上 |

→ 詳細: [ローカルコンパイルサーバー](./local-compile-server.md)

## 準備物

### 必須
- パソコン（Windows / Mac / Linux）
- Webブラウザ（Chrome、Edge推奨）
- 対応ESP32ボード
- USBケーブル（データ通信対応）
- USBドライバー（CP2102またはCH340）

### オプション
- **WiFi OTA用:** WiFiルータ、DigiCode Finder
- **BLE用:** Web Bluetooth対応ブラウザ（Chrome、Edge）
- **ローカルコンパイル用:** Docker環境

---

## 書き込みの基本フロー（USB）

1. ESP32をUSBケーブルでPCに接続
2. ブロックエディタでプログラムを作成
3. 「書き込み」→「USB」を選択
4. シリアルポートを選択
5. 書き込み完了後、ESP32が自動再起動してプログラム実行

---

## WiFi OTA / BLEを使いたい場合

初回のみ以下のセットアップが必要です：

1. USBでESP32を接続
2. 左メニュー「**ファームウェア書き込み**」→「INSTALL」を実行
3. （WiFi OTAの場合）「WiFi設定」でSSIDとパスワードを設定
4. 以降はUSB接続なしでWiFi OTA / BLEで書き込み可能

→ 詳細: [OTA設定ガイド](./05-ota-guide.md)

---

## よくある失敗と対処

| 症状 | 原因 | 対処 |
|------|------|------|
| ポートが表示されない | USBドライバー未インストール | CP2102またはCH340ドライバーをインストール |
| 書き込みエラー | USBケーブルが充電専用 | データ通信対応ケーブルに交換 |
| 書き込み中にタイムアウト | 接続不安定 | 別のUSBポート、別のケーブルを試す |
| BOOTモードに入れない | ボード固有の手順 | BOOT/BOOTSTRAPボタンを押しながら書き込み開始 |
| WiFi OTAで検出されない | WiFi未接続またはFW未書き込み | USB接続でファームウェア書き込みとWiFi設定を確認 |
